## 三级优先级

配置值按以下优先级顺序确定（从高到低）：

1. **`~/.config/fp/config.json`**（最高）— 项目根目录下的 JSON 配置文件
2. **环境变量** — 系统环境变量（与配置键同名）
3. **代码默认值**（最低）— 在 `config.py` 中硬编码

### `_value()` 实现逻辑

```python
def _value(key: str, default: Any = None) -> Any:
    """三优先级取值: JSON > 环境变量 > default"""
    # 1. JSON 中显式指定且不为 null
    if key in _json_cfg and _json_cfg[key] is not None:
        return _json_cfg[key]
    # 2. 环境变量（带类型转换）
    env_val = os.getenv(key)
    if env_val is not None:
        # 浮点型
        if key in ("TEMPERATURE",):
            return float(env_val)
        # 整型
        if key in ("MAX_TOKENS", "MAX_ITERATIONS", "TIMEOUT",
                    "RETRY_COUNT", "MAX_CONTEXT_TOKENS"):
            return int(env_val)
        return env_val
    # 3. 返回默认值
    return default
```

**类型转换**：环境变量值总是字符串，`_value()` 根据配置键自动转换为 `float` 或 `int`。

**注意**：配置键名使用**全大写 + 下划线**风格（如 `LLM_API_KEY`），与环境变量名一致。

## 配置项一览表

| 配置键 | 环境变量名 | 类型 | 默认值 | 说明 |
|--------|-----------|------|--------|------|
| `LLM_API_KEY` | `LLM_API_KEY` | str | `""` | API 密钥。未设置时启动会给出警告 |
| `LLM_API_BASE_URL` | `LLM_API_BASE_URL` | str | `"https://api.deepseek.com/v1"` | API 基础地址 |
| `LLM_MODEL` | `LLM_MODEL` | str | `"deepseek-v4-flash"` | 模型名称 |
| `TEMPERATURE` | `TEMPERATURE` | float | `0.8` | LLM 采样温度 |
| `MAX_TOKENS` | `MAX_TOKENS` | int | `32768` | 单次 LLM 调用的最大 tokens |
| `TIMEOUT` | `TIMEOUT` | int | `300` | LLM 请求超时秒数 |
| `RETRY_COUNT` | `RETRY_COUNT` | int | `3` | LLM 请求重试次数 |
| `MAX_ITERATIONS` | `MAX_ITERATIONS` | int | `50` | 工具调用最大循环次数 |
| `MAX_CONTEXT_TOKENS` | `MAX_CONTEXT_TOKENS` | int | `8000` | 上下文裁剪阈值（超过时移除最早非 system 消息） |
| `MEMORY_MAX_HISTORY` | `MEMORY_MAX_HISTORY` | int | `100` | 长期记忆最大条数 |

### Python 代码中的访问方式

```python
from fp_core import config

# 所有配置项作为模块级常量直接访问
api_key = config.LLM_API_KEY
base_url = config.LLM_API_BASE_URL
model = config.LLM_MODEL
temperature = config.LLM_TEMPERATURE
max_tokens = config.LLM_MAX_TOKENS
timeout = config.LLM_TIMEOUT
retry = config.LLM_RETRY_COUNT
max_iter = config.MAX_ITERATIONS
max_ctx = config.MAX_CONTEXT_TOKENS
max_hist = config.MEMORY_MAX_HISTORY
```

## 路径配置（固定值）

以下路径为**硬编码常量**，在 `config.py` 启动时基于项目根目录计算，不通过配置文件修改。

```python
import os

PROJECT_ROOT = os.path.dirname(os.path.abspath(__file__))  # agent/

PROMPTS_DIR = os.path.join(PROJECT_ROOT, "prompts")     # prompts/
SKILLS_DIR  = os.path.join(PROJECT_ROOT, "skills")      # skills/
DATA_DIR    = os.path.join(PROJECT_ROOT, "data")         # data/
SESSIONS_DIR = os.path.join(DATA_DIR, "sessions")        # data/sessions/
MEMORY_DIR   = os.path.join(DATA_DIR, "memory")          # data/memory/
TASKS_FILE   = os.path.join(DATA_DIR, "tasks.json")      # data/tasks.json
```

| 路径常量 | 值 | 用途 |
|---------|-----|------|
| `PROMPTS_DIR` | `项目根/prompts/` | 系统提示词模板目录 |
| `SKILLS_DIR` | `项目根/skills/` | 技能文件目录 |
| `DATA_DIR` | `项目根/data/` | 数据根目录 |
| `SESSIONS_DIR` | `项目根/data/sessions/` | 会话文件存储目录 |
| `MEMORY_DIR` | `项目根/data/memory/` | 长期记忆存储目录 |
| `TASKS_FILE` | `项目根/data/tasks.json` | 任务追踪数据文件 |

## 显示样式配置

通过 `~/.config/fp/config.json` 的 `display_styles` 和 `display_truncation` 字段配置终端输出样式。

### display_styles

为每种输出类型定义颜色和字体效果：

```json
{
    "display_styles": {
        "info":         {"color": "green"},
        "error":        {"color": "bright_red", "bold": true},
        "warning":      {"color": "yellow"},
        "yellow_bold":  {"color": "yellow", "bold": true},
        "llm_thought":  {"color": "magenta", "dim": true},
        "llm_tool":     {"color": "yellow"}
    }
}
```

| 样式名 | 默认颜色 | 用途 |
|--------|---------|------|
| `info` | green | 操作反馈/成功消息 |
| `item` | 无（默认色） | 列表子条目 |
| `hint` | 无（默认色） | 引导/用法提示 |
| `error` | bright_red + bold | 错误消息 |
| `warning` | yellow | 警告消息 |
| `yellow_bold` | yellow + bold | 强调信息 |
| `llm_thought` | magenta + dim | LLM 思考过程 |
| `llm_tool` | yellow | 工具调用信息 |

### display_truncation

```json
{
    "display_truncation": {
        "tool_result": 500,
        "code_block": 200
    }
}
```

截断长度设置为 `-1` 表示不截断。截断时在末尾追加 `… <+N chars>` 标记。

### 支持的颜色名称

`black`, `red`, `green`, `yellow`, `blue`, `magenta`, `cyan`, `white`, `bright_red`, `bright_green`, `bright_yellow`, `bright_cyan`, `bright_white`, `default`（无颜色）。
