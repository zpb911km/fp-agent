# 项目概览

---

## 定位

**Five Pebbles** 是一个 CLI/WebUI/ACP 多模式交互式 AI Agent，灵感来自游戏 *Rain World* 中的同名迭代器角色。它是一个半生物人工智能的工程化实现——冷静、理性、逻辑至上，同时具备自我迭代的能力。

核心特性：

- **LLM 对话** — 通过自实现的 OpenAI HTTP 客户端与大语言模型交互，无需官方 SDK
- **工具调用** — 支持 Function Calling 循环执行，Agent 自主决定调用时机
- **插件系统** — 生命周期钩子驱动，支持 observe/transform 两种模式，可拦截/修改/守卫流程
- **会话管理** — 基于 JSONL 文件的持久化方案，支持多会话切换、续聊、摘要生成
- **自我修改** — Agent 可以修改自身源代码并测试验证，在安全前提下突破常规
- **多模式支持** — 通过 `--mode` 参数在 CLI / WebUI / ACP 三种入口间切换

设计哲学：

> *"像卵石被水流打磨一样，通过反复试错逼近最优解。"*

## 技术栈

| 依赖 | 版本要求 | 用途 |
|------|----------|------|
| Python | ≥3.10 | 运行环境，全异步 |
| asyncio | 标准库 | 异步事件循环 |
| httpx | ≥0.24 | HTTP 客户端，替代 openai SDK |
| prompt_toolkit | ≥3.0 | CLI 交互增强（历史、补全、多行输入） |
| fastapi + uvicorn | — | WebUI 模式（可选安装 `fp[webui]`） |
| websockets | — | WebUI WebSocket 通信 |
| rich | 可选 | 富文本输出（如果安装则自动使用） |
| wcwidth | 可选 | 宽字符宽度计算（中文截断对齐） |

Python 标准库依赖：`json`、`os`、`re`、`sys`、`datetime`、`inspect`、`contextlib`、`collections`、`functools`、`enum`、`argparse`。

**无第三方 LLM SDK 依赖** —— LLM 通信完全通过自实现的 `fp_core/core/llm_client.py` 完成，使用 httpx.AsyncClient 直调 OpenAI 格式 API。

## 快速启动

```bash
# 1. 安装根包（自动安装 fp, fp-core, fp-terminal）
pip install -e .

# 2. 安装 WebUI 支持（可选）
pip install -e ".[webui]"

# 3. 安装 ACP 支持（可选）
pip install -e ".[acp]"

# 4. 配置 API Key
#    方式 A：修改 ~/.config/fp/config.json
#    方式 B：设置环境变量
export LLM_API_KEY="sk-your-key-here"

# 5. 运行（CLI 模式）
python -m fp

#    WebUI 模式
python -m fp --mode webui

#    ACP 模式
python -m fp --mode acp
```

首次启动时自动生成 `~/.config/fp/config.json`。若不配置 API Key，启动时会给出明确提示。

CLI 支持的内置命令（以 `/` 开头）：

| 命令 | 作用 |
|------|------|
| `/exit` 或 `/quit` | 退出 Agent（自动保存会话摘要） |
| `/clear` | 清空当前会话上下文 |
| `/sessions` 或 `/session list` | 列出所有会话 |
| `/session switch <sid>` | 切换到指定会话 |
| `/session compact` | 压缩当前会话 |
| `/back` | 返回上一条消息 |
| `/tools` | 列出可用工具 |
| `/reload` | 热重载所有插件和工具 |
| `/cancel` | 取消当前 LLM 请求（Ctrl+C 等效） |
| `/model` | 切换/查看当前模型 |
| `/history` | 查看历史操作记录 |
| `/fork` | 从当前对话分歧出新会话 |
| `/help` | 显示帮助信息 |

## 与 Rain World 的关联

项目中的命名和设计灵感来自游戏 *Rain World*：

- **Five Pebbles**（五块卵石）— 游戏中的一个迭代器，拥有庞大的处理能力和独特个性
- **迭代器（Iterator）** — 游戏中的半生物超级计算机，被建造来解决"大问题"（The Great Problem）
- **自我修改** — 游戏中迭代器可以修改自己的代码，本项目的自修改（self-modification）机制正是此概念的工程实现
- **卵石打磨** — 项目口号"像卵石被水流打磨一样"呼应了 Five Pebbles 的身份和不断迭代优化的设计理念

```
python -m fp
      │
      ▼
  fp/main.py ─── argparse ───┬── --mode cli ──→ fp_cli/main.py ──→ fp_core Agent

                                                      │

                                            ┌─────────────────────┐
                                            │    fp_core Agent    │
                                            │                     │
                                            │  ┌───────────────┐  │
                                            │  │  Lifecycle    │  │
                                            │  │  (19 hooks)   │  │
                                            │  └──────┬────────┘  │
                                            │         │           │
                                            │  ┌──────▼────────┐  │
                                            │  │  ToolExecutor │  │
                                            │  │  ToolRegistry │  │
                                            │  │  + plugins    │  │
                                            │  └──────┬────────┘  │
                                            │         │           │
                                            │  ┌──────▼────────┐  │
```

## 三类扩展对比

Five Pebbles 提供三种可扩展机制，各有不同定位和加载方式：

| | Plugin | Command | Tool |
|---|---|---|---|
| **中文称谓** | 插件（生命周期钩子） | 命令（斜杠输入） | 工具（函数调用） |
| **触发方式** | 生命周期事件自动触发 | 用户输入 `/xxx` | LLM 主动发起 Function Call |
| **能干什么** | 监听/拦截/修改流程，如通知、审计、守卫 | 用户直接操作的快捷行为 | LLM 自主调用的具体操作 |
| **文件格式** | `.py`，继承 `Plugin` 基类 | `.py`，导出 `name`/`execute` | `.py`，导出 `PLUGIN_DEFINITION`/`execute` |
| **加载目录** | `fp_core/plugins/` + 用户目录 | `fp_core/commands/` + 用户目录 | `fp_core/tools/core.py` + `fp_core/tools/plugins/` |
| **是否影响 LLM 上下文** | ❌ 不影响（observe 型）/ ✅ 可修改（transform 型） | ❌ 仅在终端侧执行 | ✅ 工具结果进入对话上下文 |
| **覆写机制** | 同名插件覆写 | 同名命令覆写 | 同名工具覆写 |
| **热重载** | ✅ 运行时 `scan()` | ✅ 模块重导入 | ✅ 运行时 `_load_plugins()` |
| **是否可用户扩展** | ✅ `~/.local/share/fp/plugins/` | ✅ 用户命令目录 | ✅ `~/.local/share/fp/tools/` |

> 三种扩展遵循**统一的文件命名约定**（文件即开关），详见 [文件命名约定.md](文件命名约定.md)。
