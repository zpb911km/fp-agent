## Agent 类 (`packages/fp-core/src/fp_core/core/agent.py`)

## IOChannel 体系

IO 通道抽象层用于解耦 CLI / WebUI / REST 的输入输出，使命令和 Agent 内部方法不直接依赖 `input()` 或 `display` 模块。

### 类层级

```
IOChannel（抽象基类）
 ├── CLIIO          — 终端交互，使用 input() + display 模块
 ├── WebSocketIO   — WebSocket 通道，通过 EventBus 推送输出，支持双向交互
 └── RestIO         — 静默通道，ask() 返回空字符串（单次请求-响应模式）
```

### contextvars 绑定机制

`io` 参数不再以实例变量方式存储，而是通过 `contextvars.ContextVar` 绑定到当前 asyncio Task：

- `_current_io.set(io)` 在 `process()` 入口调用 → 绑定到当前 asyncio Task
- `_current_io.reset(token)` 在 `finally` 块中恢复 → 保证不泄漏
- `agent.io` 属性优先返回 context var 覆盖值，无覆盖时 fallback 到 `__init__` 注入的默认通道

这样设计的目的是**防并发竞态**：当多个协程并发访问同一个 Agent 实例时，每个协程通过 contextvars 获得自己绑定的 IO 通道，互不干扰。

⚠️ **跨 Task 隔离**：`contextvars` 绑定到创建它的 Task，不会自动传播到子 Task。如果子 Task 需要访问正确的 IO 通道，必须使用 `Agent.make_io_context()` 手动传播。

### 各通道特点

| 通道 | ask() | say/info/error | 适用场景 |
|------|-------|----------------|----------|
| **CLIIO** | `run_in_executor(input())` | 终端着色输出 | CLI 终端 |
| **WebSocketIO** | 发 ask 事件 + Future 等待 | EventBus 推送 | WebUI / ACP |
| **RestIO** | 返回 `""`（静默取消） | 空操作 | REST API |

## Response 对象

```python
@dataclass
class Response:
    content: str = ""
    metadata: dict = field(default_factory=dict)
    error: str | None = None
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `content` | str | 最终回复文本（纯文本，非 Markdown） |
| `metadata` | dict | 元数据（当前未使用，预留） |
| `error` | str \| None | 错误信息。非 `None` 时表示处理过程中发生了异常 |

**使用示例**：

```python
agent = Agent(resume="auto")
response = await agent.process("Hello!")
if response.error:
    print(f"Error: {response.error}")
else:
    print(response.content)
```

## LifecycleManager (`packages/fp-core/src/fp_core/core/lifecycle.py`)

事件驱动的生命周期管理器，支持同步/异步钩子，按优先级执行。

### 类签名

```python

    def __init__(self, enable_log: bool = False)
```

### 核心方法

```python
def register(
    self,
    hook: LifecycleHook,
    func: Callable,
    priority: int = 100,
    name: str | None = None,
    hook_type: str | None = None,
)
```

注册钩子函数。

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `hook` | LifecycleHook | — | 要注册的钩子枚举 |
| `func` | Callable | — | 钩子处理函数（同步或异步均可） |
| `priority` | int | `100` | 优先级，越小越先执行 |
| `name` | str \| None | `None` | 名称（用于注销/调试），默认取函数名 |
| `hook_type` | str \| None | `None` | `"observe"` / `"transform"`。`None`=自动推断 |

```python
def unregister(self, hook: LifecycleHook, name: str) -> bool
```

注销指定钩子。成功返回 `True`，未找到返回 `False`。

```python
async def emit(
    self,
    hook: LifecycleHook,
    context: HookContext | None = None,
    **kwargs,
) -> HookContext
```

异步触发钩子。返回最终的 `HookContext`（可能被 transform 型钩子修改）。调用方应检查返回的 `context.data` 获取插件修改后的值。

**emit 调用约定**：

1. **kwargs → context.data 合并**：所有额外的 `**kwargs` 会自动合并到 `context.data` 中（若传入 `data=dict()` 会展开合并而非覆盖嵌套），钩子处理函数通过 `context.data[key]` 或 `**context.data` 解包访问。
2. **返回值合并**：若某个钩子函数返回了 `HookContext` 实例，`emit` 会用返回值替换当前 context，实现链式数据传递。
3. **异常隔离**：`observe` 型钩子的异常被隔离（仅记录日志），`transform` 型钩子的异常会直接向上传播并标记 `context.stop_propagation = True`。
4. **停止传播**：通过 `context.stop_propagation = True` 可提前终止后续钩子执行，用于守卫拦截场景。

### @hook 装饰器

```python
from fp_core.core.lifecycle import LifecycleHook, LifecycleManager, hook

lifecycle = LifecycleManager()

@hook(lifecycle, LifecycleHook.ON_BEFORE_LLM_CALL, priority=50)
async def my_llm_guard(context, messages, tools, **kwargs):
    """在 LLM 调用前检查消息内容"""
    for msg in messages:
        if msg["role"] == "user" and len(msg["content"]) > 10000:
            context.data["cancelled"] = True
            context.data["cancel_reason"] = "消息过长"
    return context
```

**装饰器定义**（示意）：

```python
def hook(lifecycle: LifecycleManager, hook_enum: LifecycleHook, priority: int = 100, name: str | None = None):
    def decorator(func):
        lifecycle.register(hook_enum, func, priority=priority, name=name)
        return func
    return decorator
```

### 完整示例

```python
from fp_core.core.lifecycle import LifecycleManager, LifecycleHook, HookContext

lifecycle = LifecycleManager(enable_log=True)

# observe 型钩子：仅观察，不修改
def on_message(ctx: HookContext, **kwargs):
    print(f"[Observe] 用户说了: {kwargs.get('filtered_content', '')[:50]}")

lifecycle.register(LifecycleHook.ON_MESSAGE_RECEIVED, on_message, priority=200)

# transform 型钩子：可修改数据
async def on_filter(ctx: HookContext, **kwargs):
    content = kwargs.get("filtered_content", "")
    if "敏感词" in content:
        ctx.data["blocked"] = True
        ctx.data["block_reason"] = "包含敏感内容"
    return ctx

lifecycle.register(LifecycleHook.ON_MESSAGE_FILTER, on_filter, priority=50)
```
