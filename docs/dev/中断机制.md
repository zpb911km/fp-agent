# 中断机制

中断机制处理用户通过 `Ctrl+C` (SIGINT) 发出的取消请求，确保中断可以被及时检测并优雅处理。

---

## 架构

```
用户 Ctrl+C  →  SIGINT  →  信号处理器
```

### 信号处理流程

1. 操作系统捕获 `SIGINT`
2. 调用 `_raw_sigint_handler()`
3. 该处理器同时设置两种中断标志（双重检查）
4. Agent 在关键检查点检测标志并抛 `asyncio.CancelledError`

### 跨平台回退

- **Unix**：通过 `asyncio.all_tasks().cancel()` 向所有异步任务注入 `CancelledError`
- **跨平台保底**：同时设置全局 `_interrupted_flag`，供同步代码轮询检查

---

## 双重检查机制

同时设置**异步取消**和**全局标志**，确保无论当前执行路径是 async 还是 sync 都能响应中断。

```python
import asyncio, signal

_interrupted_flag = False

def _raw_sigint_handler(signum, frame):
    global _interrupted_flag
    _interrupted_flag = True          # ① 全局标志（同步代码检查）
    # ② 异步取消
    try:
        loop = asyncio.get_running_loop()
        for task in asyncio.all_tasks(loop=loop):
            task.cancel()
    except RuntimeError:
        pass  # 没有运行中的事件循环

# 注册
signal.signal(signal.SIGINT, _raw_sigint_handler)
```

### Agent 检查点

```python
class Agent:
    def _check_interrupted(self):
        """在两个入口处检查中断"""
        global _interrupted_flag
        if _interrupted_flag:
            _interrupted_flag = False
            raise asyncio.CancelledError("用户中断")
        # asyncio 的 task.cancel() 会在 await 处自动抛 CancelledError
```

## 检查点位置

中断检查分布在三个关键层级：

| 层级 | 位置 | 机制 |
|------|------|------|
| **LLM 调用** | `packages/fp-core/src/fp_core/core/llm.py` 的 `stream_chat()` | httpx 请求可取消（底层 socket 读中断即抛异常） |
| **工具执行** | `tools/__init__.py` 的 `execute()` | bash 子进程可 kill（通过 `asyncio.create_subprocess_exec` 终止） |
| **主循环** | `packages/fp-core/src/fp_core/core/agent.py` 的 `process()` | while 循环每次迭代前调用 `_check_interrupted()` |

## 中断恢复策略

### LLM 调用中断

```
保留已输出内容（LLMStreamer.end(interrupted=True)）
标记计划调用的工具【未执行】
→ 中断后用户可执行 /resume 继续
```

### 工具执行中断

```
当前正在执行的工具 → 标记为失败（返回错误信息）
剩余未执行的工具 → 标记为未执行
→ 直接退出工具循环，不继续调度
```

## _processing 标志

`Agent` 在 LLM 调用前后设置 `_processing` 标志，用于防止中断在错误时机触发清理：

```python
class Agent:
    async def _call_llm(self, ...):
        self._processing = True       # 进入 LLM 调用前
        try:
            result = await llm.stream_chat(...)
        finally:
            self._processing = False  # 调用后恢复
```

- `_processing = True` 期间，信号处理器记录中断但不立即清理
- `_processing = False` 后，主循环在下一次迭代时处理中断
