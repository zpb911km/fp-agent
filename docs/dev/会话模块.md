# 会话模块

---

## SessionManager (`core/session.py`)

会话管理器负责会话的持久化。它**不持有上下文消息**——只负责读写文件。上下文状态的所有权在 `ConversationState`。

### 类签名

```python
class SessionManager:
    def __init__(self, resume: str | None = None)
```

| 参数 | 类型 | 值 | 行为 |
|------|------|----|------|
| `resume` | `None` / `False` | — | 创建新会话（默认） |
| `resume` | `True` / `"auto"` | — | 续最近会话（按 `updated` 时间） |
| `resume` | `"s_xxx"` | — | 续指定会话 ID（如果存在） |

内部逻辑：先归一化 `resume` 值（`False`→`None`，`True`→`"auto"`），再按分支处理。

### 属性

```python
@property
def session_id(self) -> str
```

当前会话 ID，格式如 `s_260606_160000000000`。

### 会话管理方法

```python
def list_sessions(self) -> dict[str, dict[str, Any]]
```

扫描 `data/sessions/` 目录，返回所有会话的 meta 信息字典：`{sid: {meta_fields}}`。

```python
def switch_session(self, sid: str) -> bool
```

切换到指定会话。目标会话必须存在。返回是否成功。

```python
def create_session(self) -> str
```

创建新会话并切换到它。返回新会话 ID。

```python
def resume_latest(self) -> bool
```

尝试续最近会话（按 `updated` 字段）。如果存在历史会话则切换到它并返回 `True`；否则创建新会话并返回 `False`。

```python
def delete_session(self, sid: str) -> bool
```

删除指定会话文件。不能删除当前会话。返回是否成功。

```python
def clear_session_file(self)
```

清空当前会话文件（重置为默认 meta，删除所有历史消息）。

### 消息读写方法

```python
def save_message(self, role: str, content: str, **kwargs)
```

追加一条消息到文件末尾。自动更新文件内嵌的 meta（`message_count++`, `updated`）。支持 `tool_calls`、`tool_call_id`、`reasoning_content` 等附加字段。

```python
def save_context(self, context: list[dict[str, Any]])
```

将完整上下文**重写**到文件（原子写入：先写 `.tmp` 再 `os.replace`）。跳过第一条 `system` 消息（由 `load_context` 重新加载）。更新 `message_count` 和 `updated` 时间戳。

```python
def load_context(self, system_prompt: str) -> list[dict[str, Any]]
```

加载会话文件中的所有历史消息。返回的列表第 0 项是传入的 `system_prompt`（作为 `{"role": "system", "content": system_prompt}`），后续项是从文件读取的历史消息（跳过第 1 行的 meta）。

```python
def update_meta(self, sid: str | None = None, **kwargs)
```

更新指定会话的内嵌 meta 字段。自动更新 `updated` 时间戳。如果 `sid` 为 `None` 则更新当前会话。

## 会话文件格式

每个会话存储为 **一个 `.jsonl` 文件**，位于 `data/sessions/` 目录下。

### 文件结构

```json
// 第 1 行 — meta 信息（JSON 单行）
{"__meta__": true, "id": "s_260606_160000000000", "created": "2026-06-06 16:00:00", "updated": "2026-06-06 16:30:00", "summary": "用户问题概要", "message_count": 12}

// 第 2 行起 — 消息（按时间正序，最新在末尾）
{"role": "user", "content": "Hello"}
{"role": "assistant", "content": "Hi! How can I help?"}
{"role": "user", "content": "查询天气"}
{"role": "assistant", "content": "", "tool_calls": [{"id": "call_xxx", "type": "function", "function": {"name": "get_weather", "arguments": "{\"city\": \"北京\"}"}}]}
{"role": "tool", "content": "{\"temperature\": 25}", "tool_call_id": "call_xxx"}
{"role": "assistant", "content": "北京当前 25°C"}
```

### 格式约定

- **第 1 行** — 固定为 meta 行，包含 `__meta__: true` 标记
- **第 2+ 行** — 消息行，不含 `system`（system prompt 由 PromptBuilder 在加载时重新创建）
- **正序** — 最早的消息在前，最新消息在末尾
- **单文件** — 没有独立的 meta 文件或 `_current` 文件
- **system 不入文件** — 第一条 system 消息在 `save_context` 时被跳过

### meta 字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `__meta__` | bool | 固定为 `true`，用于标识 meta 行 |
| `id` | str | 会话 ID |
| `created` | str | 创建时间 `"YYYY-MM-DD HH:mm:ss"` |
| `updated` | str | 最后更新时间 |
| `summary` | str | 会话摘要（取自最后一条用户消息的前 20 字符） |
| `message_count` | int | 消息总数（不含 system） |

### 加载规则

```python
def load_context(self, system_prompt: str) -> list[dict]:
    context = [{"role": "system", "content": system_prompt}]  # 第 0 项
```

## 会话 ID 生成

```python
def _generate_sid() -> str:
    base = datetime.now().strftime("s_%y%m%d_%H%M%S%f")  # 微秒级精度
    sid = base
    for i in range(100):
        if not os.path.exists(_session_path(sid)):
            return sid
        sid = f"{base}_{i}"  # 冲突时加自增后缀
    import random
    return f"{base}_{random.randint(1000, 9999)}"
```

**格式**：`s_YYMMDD_HHMMSSffffff`（微秒级，6 位微秒）

**冲突处理**：如果同一微秒内生成重复 ID，依次尝试 `_0`、`_1`……`_99` 后缀。极端情况下加随机后缀。

**文件名**：`s_260606_160000000000.jsonl`

**文件模式匹配**：

```python
SID_PATTERN = re.compile(r"^s_\d{6}_\d{12,}\.jsonl$")
```
