# 工具概览

## 核心工具（直接绑定）

以下工具直接绑定到系统核心，无需通过技能系统加载，始终可用。

| 工具名 | 说明 |
|--------|------|
| `bash` | 执行任意 shell 命令，支持管道/重定向，超时 300 秒 |
| `read_file` | 读取文件内容，支持指定行范围和行数限制 |
| `write_file` | 创建新文件或覆盖已有文件 |
| `edit_file` | 对文件进行精确字符串替换（只替换首次出现的位置） |

这些工具是 Agent 与操作系统交互的基础设施，所有文件操作和命令执行都依赖它们。

---

## 插件工具

以下工具通过技能/插件系统提供，可按需调用。

| 工具名 | 说明 |
|--------|------|
| `file_fingerprint` | 使用 file 和 strings 提取文件特征，判断类型 |
| `memory_read` | 读取长期记忆（空查询列出全部，关键词搜索） |
| `memory_save` | 保存一条长期记忆到持久化存储 |
| `python` | 执行 Python 代码，适合复杂数据处理和算法验证 |
| `subagent` | 派遣子 Agent 执行独立任务，节省主上下文 Token |
| `task_create` | 创建待办任务，用于多步骤工作跟踪 |
| `task_update` | 更新已有任务的状态 |
| `task_list` | 列出所有任务及其当前状态 |
| `task_clear` | 清除所有已完成的任务记录 |
| `web_search` | 快速搜索，返回标题+链接+摘要列表 |
| `web_fetch` | 抓取并解析网页内容，返回纯文本 |

## 工具调用流程

整个工具调用的生命周期如下：

```
用户输入
    │
    ▼
LLM 思考（分析意图、决定调用的工具和参数）
    │
    ▼
调用工具（系统执行对应的工具函数）
    │
    ▼
结果返回 LLM（工具输出作为上下文的一部分）
    │
    ▼
LLM 生成最终回复（基于工具结果组织语言）
    │
    ▼
回复用户
```

### 终端显示

工具调用过程会实时在终端显示，以 `🛠️` 图标标识：

```
🛠️ 正在调用工具: bash(command="ls -lh data/sessions/")
🛠️ 工具执行完毕（耗时 0.32s）
```

这使得整个过程对用户透明，可以看到 Agent 每一步的操作和耗时。
