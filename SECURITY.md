# 安全策略

## ⚠️ 重要声明

**请先阅读 [README 中的安全警告](./README.md#-安全警告--这是五块卵石)。**

本项目是一个自举（self-bootstrapping）AI Agent 系统：**全部代码由 AI Agent 自己编写、自己修改、自己运行。** 它从根上没有安全设计——没有沙箱、无身份认证、无审核流程。这不是疏忽，这是本质。

## 报告的漏洞可能不会被修复

考虑到项目的自举性质：

- **代码由 AI 生成**— 人类维护者并不完全理解每一行代码的含义
- **安全修复可能被 AI 后续自修改覆盖**— 修复了，下次自迭代又改回去了
- **"安全"与"自举"互斥**— 限制 AI 修改自身的能力，就等于阉割核心功能

因此，请不要期望报告的安全漏洞会得到及时的修补。

如果你发现了涉及真实用户数据泄露、或可能对他人造成实质性损害的漏洞，我们仍然希望了解它：

3. 建议的缓解措施（如果有）

## 建议的防护措施

如果要在任何接近生产环境的地方使用此系统，请至少做到：

1. **隔离运行** — 使用 Docker 容器或虚拟机，限制文件系统和网络访问
2. **审计日志** — 记录 Agent 执行的所有 bash 命令和文件修改
3. **定期备份** — 版本控制你的所有配置和技能文件
4. **不要暴露端口** — WebUI 和 ACP 永远绑定 `127.0.0.1`
5. **审查 AI 输出** — 永远不要盲目信任 AI 生成的 bash 命令
