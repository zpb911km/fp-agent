# 命令参考

Five Pebbles CLI 提供 14 个斜杠命令（含别名共 16 个）。所有命令以 `/` 开头，支持 Tab 补全。

---

## 命令速查表

| 命令 | 别名 | 功能 |
|------|------|------|
| `/help` | `/?` | 显示帮助信息 |
| `/exit` | `/quit` | 正常退出，LLM 生成总结，显示退出面板 |
| `/exit!` | — | 核弹级退出，删除当前会话并立即退出 |
| `/clear` | — | 清空当前会话历史 |
| `/reset` | — | 重置上下文到初始状态 |
| `/back` | — | 回退到对话历史某处 |
| `/history` | — | 查看当前会话历史摘要 |
| `/fork` | — | 基于当前上下文创建新会话并切换 |
| `/compact` | — | 压缩对话历史，节省上下文空间 |
| `/resume` | — | 切换/删除历史会话 |
| `/reload_skills` | — | 热重载技能 |
| `/remove_skill` | — | 移除指定技能 |
| `/skills` | — | 列出已加载技能及其描述 |
| `/model` | — | 显示模型和路径信息 |
| `/memory` | — | 列出持久化记忆 |
| `/session` | — | 显示当前会话 ID |

## `/help` （`/?`）

**功能**：显示所有可用命令及其简短描述。

以排序列表形式输出，每条命令后附有功能说明。此命令的输出也是 Tab 补全菜单的描述信息来源。

输出示例：

```
可用命令:
  /back         回退到对话的某个历史时刻。用法: /back list 查看列表, /back <N> 直接回退
  /clear        清空当前会话
  /compact      压缩对话历史
  /exit         退出程序
  /exit!        核弹级退出：删除当前会话、不留痕迹
  /fork         基于当前上下文新建会话
  /help         显示此帮助
  /history      查看当前对话历史
  /memory       查看持久化记忆
  /model        显示当前模型配置
  /reload_skills  热重载技能
  /remove_skill   删除指定技能
  /reset        重置上下文（保留系统提示）
  /resume       切换/删除历史会话。用法: /resume list, /resume latest, /resume <sid>, /resume delete <sid>
  /session      显示当前会话信息
  /skills       列出所有技能
```

---

## `/exit` （`/quit`）

**功能**：正常退出程序。

触发完整关闭流程：

1. 从对话历史中提取最后一条用户消息作为会话摘要
2. 更新会话元数据（摘要标签、消息数等）
3. 保存当前上下文到会话文件
4. 显示退出统计面板
5. 优雅关闭 Agent

### 退出面板示例

```
╔══════════════════════════════════════════════════════╗
║  📂  会话结束                                        ║
║                                                     ║
║  总结: 你好请介绍一下你自己                            ║
║  文件: session_abc123.jsonl                          ║
║                                                     ║
║  📊  统计信息                                        ║
║                                                     ║
║  模型: deepseek-v4-flash                             ║
║  消息: 12 条                                        ║
║  创建: 2026-06-11 20:15:33                          ║
║  耗时: 0:03:45                                      ║
╚══════════════════════════════════════════════════════╝

👋  再见！
```

## `/exit!`

**功能**：核弹级退出，删除当前会话文件，不留痕迹。

该命令不可逆，执行后：

1. 标记核弹退出模式
2. 立即删除当前会话的 JSONL 文件
3. 退出面板**不显示**（因为文件已被删除）
4. 不会生成会话摘要

适宜场景：测试、调试、或不想在系统里留下任何对话痕迹。

> ⚠️ 使用前请确认你不需要保留当前对话。文件删除后无法恢复。

---

## `/clear`

**功能**：清空当前会话历史。

- 保留系统提示词（system prompt）
- 保留会话 ID
- 仅删除用户消息、AI 回复和工具调用记录
- 等价于重置上下文，但不清除会话元数据

输出：`🧹 当前会话已清空`

---

## `/reset`

**功能**：重置上下文到初始状态。

- 重新加载系统提示词
- 重新加载技能列表
- 清空所有对话消息
- 与会话 ID 无关，只是上下文层面的重置

与 `/clear` 的区别：`/reset` 更彻底，会重建系统提示词和技能上下文；`/clear` 只清空消息列表。

输出：`🔄 上下文已重置`

---

## `/back`

**功能**：回退到对话历史的某个时刻，删除该位置之后的所有消息。

### 用法

```
/back list              查看历史消息列表
/back <N>               回退到第 N 条消息处（删除后续消息）
```

### 交互示例

```
你 > /back list
📜 对话历史（共 5 条消息，使用 /back <N> 回退）:
  [  1] 👤 用户: 帮我写一个 Python 脚本
  [  2] 🤖 AI: 好的，这里是脚本...
  [  3] 👤 用户: 能加一个错误处理吗？
  [  4] 🤖 AI: 好的，我添加了 try/except...
  [  5] 🔧 工具: 脚本运行结果: OK

你 > /back 3
（已删除消息 4-5，上下文回到消息 3 之后的状态）
```

消息列表按 1-based 编号，仅显示非 system 消息。`/back list` 中每条消息显示角色和内容摘要（超长自动截断）。

---

## `/history`

**功能**：查看当前会话中的对话历史摘要。

直接在终端中输出当前会话的所有非系统消息，包括角色和内容。

---

## `/fork`

**功能**：基于当前对话上下文创建一个新的会话，并自动切换到新会话。

- 原会话保持不变，带摘要标签保存
- 新会话拥有与当前会话完全相同的上下文（系统提示 + 所有历史消息）
- 后续输入进入新会话

输出：`🍴 已 fork 新会话`

---

## `/compact`

**功能**：压缩对话历史，节省上下文空间。

- 调用 LLM 将较早的消息（除最近 4 条外）压缩为一段摘要
- 压缩后的摘要替换原始消息，显著减少 Token 消耗
- 压缩操作是异步的，执行期间会显示等待反馈

输出：`📦 历史已压缩`

---

## `/resume`

**功能**：切换或删除历史会话。支持多种用法。

### 用法

```
/resume list               列出所有会话
/resume latest             切换到最新会话（按更新时间）
/resume <SESSION_ID>       切换到指定 ID 的会话
/resume delete list        列出可删除的会话
/resume delete <SESSION_ID> 删除指定会话
```

### 交互示例

```
你 > /resume list
📂 会话列表（使用 /resume <sid> 切换，/resume latest 切换到最新）:
  [1] 帮我搜索 Python 异步编程...    ( 8条, session_x1a2b3c)
  [2] 写一个排序算法...             (12条, session_y4d5e6f)  ⬅
  [3] 分析这个 ELF 文件...          ( 5条, session_z7g8h9i)

你 > /resume session_z7g8h9i
📂 已切换到会话: session_z7g8h9i
```

- 当前正在使用的会话以 `⬅` 标记
- 当前会话不可以删除
- 切换会话后上下文会重建为对应会话的状态

---

## `/reload_skills`

**功能**：热重载 `skills/` 目录下的所有技能文件。

- 不需要重启 Agent
- 重新扫描目录并加载所有 `.md` 技能文件
- 适合在 AI 修改了技能文件后立即生效

输出：`✅ 技能重载完成` 或 `❌ 技能重载失败`

---

## `/remove_skill <name>`

**功能**：从 Agent 的技能列表中移除指定技能。

- 仅从运行时移除，不删除技能文件
- 移除后可通过 `/reload_skills` 重新加载

参数：`<name>` 为技能名称（即 `.md` 文件名，不含扩展名）。

输出：`🗑️ 技能 '<name>' 已删除（如存在）`

---

## `/skills`

**功能**：列出当前已加载的所有技能及其描述。

输出格式为结构化文本列表，每行显示技能名称和功能说明。

---

## `/model`

**功能**：显示当前模型配置和关键路径信息。

输出示例：

```
模型:   deepseek-v4-flash
温度:   0.8
最大 Token: 32768
会话目录: /path/to/agent/data/sessions
```

---

## `/memory`

**功能**：列出持久化记忆系统中的所有记忆条目。

- 扫描 `data/memory/` 目录下的所有 `.md` 文件
- 每条记忆以文件名的形式列出
- 附加提示：`💡 使用 memory_read / memory_save 工具管理记忆`

输出示例：

```
📋 记忆列表:

  • user_name
  • project_goal
  • feedback_preference
```

---

## `/session`

**功能**：显示当前会话的唯一 ID。

输出示例：

```
📂 当前会话: session_a1b2c3d4e5f6
```
