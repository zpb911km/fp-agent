# 自我修改

Agent 具备修改自身源代码并验证的能力，通过严格的流程控制确保修改安全可控。

---

## 流程

自我修改遵循六步流水线：

```
① task_create 列出步骤
    ↓
② task_update(in_progress)
    ↓
③ 修改代码 (edit_file / write_file)
    ↓
④ 测试验证 (echo | python3 cli.py -m ...)
    ↓
⑤ 判断结果
   ├─ 通过 → git commit
   └─ 失败 → git checkout -- . (回滚)
    ↓
⑥ task_update(completed)
```

### 详细说明

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① 任务规划 | `task_create` | 将每次修改拆分为独立子任务，分别追踪 |
| ② 标记进行中 | `task_update(id, "in_progress")` | 告知用户当前正在进行哪一步 |
| ③ 修改代码 | `edit_file` / `write_file` | 精确字符串替换或创建新文件 |
| ④ 测试 | `echo "提示词" \| python3 cli.py -m "消息"` | 模拟用户输入测试修改效果 |
| ⑤ 判定 | 通过→提交 / 失败→回滚 | 失败时用 git 放弃所有未提交修改 |
| ⑥ 完成 | `task_update(id, "completed")` | 标记任务完成 |

---

## 测试命令格式

```bash
echo "测试提示词" | python3 cli.py -m "用户消息"
```

- `echo` 的内容作为首次输入的提示词注入到 system prompt 上下文
- `-m` 参数指定模拟的用户消息
- 观察输出是否符合预期来判定修改是否成功

---

## 回滚命令

### 完全回滚（放弃所有未提交修改）

```bash
git checkout -- .
```

### 选择性回滚

```bash
git checkout -- core/agent.py       # 仅回滚 agent.py
git checkout -- tools/core.py       # 仅回滚工具核心
git checkout -- plugins/my_plugin.py # 仅回滚某个插件
```

**注意：** `git checkout -- .` 会丢弃所有未暂存/未提交的修改，且**不可撤销**。确认回滚前应确保测试确实失败。
