# 工具系统

工具系统是 Agent 与外部世界交互的桥梁。LLM 通过 Function Calling 声明需要调用的工具，由 ToolRegistry 统一调度执行并返回结果。

---

## ToolRegistry (tools/__init__.py)

`ToolRegistry` 是全局唯一的工具注册中心，管理核心工具与插件工具的声明与执行。

**核心 API：**

```python
class ToolRegistry:
    def get_all_definitions(self) -> list[dict]:
        """返回所有工具的 OpenAI Function Calling schema"""
    async def execute(self, tool_name: str, params: dict) -> Any:
        """按名称执行工具，返回执行结果"""
```

**全局快捷函数（`from fp_core.tools import dispatch`）：**

| 函数 | 对应工具 |
|------|----------|
| `dispatch(tool_name, params)` | 通用调度 |
| `bash(command)` | 执行 shell 命令 |
| `read_file(file_path, offset, limit)` | 读取文件 |
| `write_file(file_path, content)` | 写入文件 |
| `edit_file(file_path, old_string, new_string)` | 编辑文件 |
| `python(code)` | 执行 Python |
| `web_search(query)` | 网页搜索 |

## 核心工具（不可插件化）

4 个核心工具在 `tools/core.py` 中定义，**不可通过插件系统卸载或覆盖**。

| 工具名 | 参数 | 说明 |
|--------|------|------|
| `bash` | `command: str` | 执行任意 shell 命令，超时 300s |
| `read_file` | `file_path: str, offset: int?, limit: int?` | 读取文件，支持行范围 |
| `write_file` | `file_path: str, content: str` | 创建/覆写文件 |
| `edit_file` | `file_path: str, old_string: str, new_string: str` | 精确字符串替换（首次出现） |

`tools/core.py` 中每个工具定义包含：

1. **OpenAI schema** —— 描述参数结构
2. **异步执行函数** —— `async def execute_xxx(params) -> str`
3. **统一分发入口** —— `execute_core_tool(name, params)` 负责路由

```python
# tools/core.py 结构示意
TOOL_DEFINITIONS = [
    {
        "type": "function",
        "function": {
            "name": "bash",
            "description": "执行 shell 命令",
            "parameters": {
                "type": "object",
                "properties": {
                    "command": {"type": "string", "description": "命令内容"}
                },
                "required": ["command"]
            }
        }
    },
    # ...read_file, write_file, edit_file
]

async def execute_core_tool(name: str, params: dict) -> str:
    # 路由到对应实现
```

## 插件工具开发

在 `tools/plugins/` 目录下创建 `*_plugin.py` 文件，必须导出以下内容：

| 导出名 | 类型 | 说明 |
|--------|------|------|
| `PLUGIN_DEFINITION` | `dict` | OpenAI Function Calling schema |
| `execute` | `(params: dict) -> str` | 执行函数（同步/异步均可） |

**完整示例（伪工具 `my_new_tool`）：**

```python
"""tools/plugins/my_new_tool_plugin.py"""
PLUGIN_DEFINITION = {
    "type": "function",
    "function": {
        "name": "my_new_tool",
        "description": "一个示例插件工具——将输入翻转",
        "parameters": {
            "type": "object",
            "properties": {
                "input": {
                    "type": "string",
                    "description": "要翻转的文本"
                }
            },
            "required": ["input"]
        }
    }
}

async def execute(params: dict) -> str:
    """推荐异步实现；错误时 raise Exception。"""
    text = params["input"]
    return text[::-1]  # 返回字符串
```

**规则：**
- `execute` 必须返回 `str`（LLM 接收的是字符串）
- 推荐异步（`async def`），同步会自动包装
- 发生错误时 `raise Exception`，由框架统一捕获并返回错误消息

## 自动注册流程

`ToolRegistry._load_plugins()` 的扫描逻辑：

1. 扫描 `tools/plugins/` 目录下所有 `.py` 文件
2. 过滤：跳过 `_plugin.py` 后缀不匹配的？实际检测：结尾为 `_plugin.py`
3. `importlib.import_module` 动态导入
4. 检查是否导出 `PLUGIN_DEFINITION` 和 `execute`
5. 校验 `PLUGIN_DEFINITION` 中的 `function.name` 不与核心工具及已有插件重名
6. 注册成功后将 `name → (definition, execute)` 存入 `self._plugins` 字典
