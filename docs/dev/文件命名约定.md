# 文件命名约定 —— 通用规则

本文描述的「文件即开关」命名约定，是 Five Pebbles 所有可扩展系统的**共同基础**，
统一适用于以下四类扩展：

| 扩展类型 | 目录 | 文件格式 |
|----------|------|----------|
| **Plugin**（生命周期钩子） | `fp_core/plugins/` + `~/.local/share/fp/plugins/` | `.py`，继承 `Plugin` 基类 |
| **Skill**（技能提示词） | `fp_core/skills/` | `.md` + YAML frontmatter |
| **Command**（斜杠命令） | `fp_core/commands/` + 用户目录 | `.py`，导出 `name`/`execute` |
| **Tool**（Function Call 工具） | `fp_core/tools/plugins/` | `.py`，导出 `PLUGIN_DEFINITION`/`execute` |

> 四类扩展各有独立的加载机制和接口约定，详见各自文档：
> - [插件系统.md](插件系统.md) — Plugin 生命周期钩子
> - [技能系统.md](技能系统.md) — SkillLoader 技能提示词
> - [命令系统.md](命令系统.md) — CommandRegistry 斜杠命令
> - [工具系统.md](工具系统.md) — ToolRegistry 函数调用工具

## 文件即开关

每个文件对应一个独立扩展单元。

**启用/停用不需要改配置、不碰代码——改文件名就够了。**

```
plugins/
├── notification.py         # ✅ 启用
├── weather.py              # ✅ 启用
├── something.py.disabled   # ❌ 停用（加了后缀 .disabled）
├── old_idea.py.backup      # ❌ 停用（加了后缀 .backup）
├── __pycache__/            # ⚠️ Python 缓存，自动忽略
└── base/                   # ⚠️ 基类目录，不直接加载
```

> Skills 目录同理：
> ```
> skills/
> ├── my_skill.md            # ✅ 启用
> ├── old_skill.md.disabled  # ❌ 停用
> └── draft.md.backup       # ❌ 停用
> ```

### 规则

| 文件名模式 | 行为 |
|-----------|------|
| `name.py` / `name.md` | ✅ 加载（有效扩展） |
| `name.py.任意后缀` / `name.md.任意后缀` | ❌ 跳过（停用标记） |
| `_name.py` / `_name.md` | ❌ 跳过（内部模块） |
| `base/`, `__pycache__/` | ❌ 自动忽略 |

### 使用场景

```
# 临时停用一个扩展
mv notification.py notification.py.disabled

# 恢复
mv notification.py.disabled notification.py

# 保留一份旧版本参考
mv old_plugin.py old_plugin.py.v2
```

### 为什么这样设计

1. **零配置** — 文件系统本身就是配置层
2. **无侵入** — 不改代码、不写配置文件、不碰注册表
3. **原子操作** — `mv` 命令即开关，不会出现"改了配置但忘重启"的状态不一致
4. **版本兼容** — 停用的扩展原样留在目录里，随时可恢复，不会丢失历史
