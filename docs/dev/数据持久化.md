# 数据持久化

系统维护三类持久化数据：会话记录、长期记忆、任务列表，分别存储在 XDG 兼容路径下。

---

## 会话文件格式

会话文件位于 `~/.local/share/fp/sessions/`，命名格式 `s_{uuid}.jsonl`。

### JSONL 格式细节

每行一个独立的 JSON 对象，**行内不允许有换符**。

### 关键规则

| 规则 | 说明 |
|------|------|
| 第 1 行始终是 `__meta__` | 存储会话元信息，以 `__meta__` 键标记 |
| 正序存储 | 最早的消息在前，最新在后 |
| `system` 不写入 | system prompt 不持久化到会话文件 |
| `save_context` 覆写 | 最后一条用户消息可标记 `save_context`，下次加载时作为上下文恢复 |

### 示例数据

```jsonl
{"__meta__": true, "id": "s_abc123", "model": "qwen2.5:32b", "created_at": "2026-06-10T12:00:00", "message_count": 3}
{"role": "user",    "content": "你好，请解释什么是迭代器"}
{"role": "assistant", "content": "迭代器是一种能够遍历数据集合的对象..."}
{"role": "user",    "content": "能否给我一个代码示例？", "save_context": true}
{"role": "assistant", "content": "```python\\nfor i in range(10):\\n    print(i)\\n```"}
```

**`__meta__` 行字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `__meta__` | bool | 标记该行为元信息行，固定为 `true` |
| `id` | str | 会话唯一 ID |
| `model` | str | 使用的模型名称 |
| `created_at` | str | ISO 8601 时间戳 |
| `message_count` | int | 消息总数（不含 meta） |
| `forked_from` | str? | 来源会话 ID（分叉时） |

## 记忆文件格式

长期记忆文件位于 `~/.local/share/fp/memory/`，命名格式 `{name}.md`。每个文件是一份独立的记忆记录。

### 格式：Markdown + YAML frontmatter

```markdown
---
name: user_preference
description: 用户偏好的编程语言和工具链
type: user
created: 2026-06-01
---

用户偏好 Python 和 TypeScript，使用 VS Code 作为编辑器，
熟悉 fastapi、react 和 tailwindcss。
```

### 元数据字段

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `name` | 是 | str | 记忆唯一标识名 |
| `description` | 是 | str | 一句话描述 |
| `type` | 是 | str | 类型：`user` / `project` / `feedback` / `reference` |
| `created` | 是 | str | 创建日期（ISO 8601 或 YYYY-MM-DD） |

## 任务文件格式

任务文件位于 `~/.local/share/fp/tasks.json`，存储待办任务列表。

### 格式：JSON 数组

```json
[
  {
    "id": 1,
    "subject": "分析日志文件",
    "description": "扫描 /var/log/syslog 提取错误信息",
    "status": "completed",
    "created_at": "2026-06-10T14:30:00"
  },
  {
    "id": 2,
    "subject": "编写单元测试",
    "description": "为 tools/core.py 添加 pytest 测试",
    "status": "in_progress",
    "created_at": "2026-06-10T15:00:00"
  },
  {
    "id": 3,
    "subject": "重构 display 模块",
    "description": "将样式配置抽取到 config.json",
    "status": "pending",
    "created_at": "2026-06-10T16:00:00"
  }
]
```

### 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 自增 ID |
| `subject` | str | 任务标题 |
| `description` | str | 任务详细描述 |
| `status` | str | `pending` / `in_progress` / `completed` |
| `created_at` | str | ISO 8601 时间戳 |

### 操作

通过 `task_create`、`task_update`（修改状态）、`task_clear`（删除已完成）、`task_list`（列出全部）四个工具操作任务文件。
