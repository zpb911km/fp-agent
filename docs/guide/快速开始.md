# 快速开始

**Five Pebbles（五块卵石）** 是一个以 LLM 为推理核心、工具为执行手臂的自主 Agent 框架。本文档带你从零开始，逐步了解安装、初始化、内置系统和插件扩展的全貌。

---

## 📦 一、安装

### 从 PyPI 安装

```bash
# 基础安装（核心引擎 + CLI 终端）
pip install fp-agent

# 完整安装（核心 + CLI + Web 界面 + ACP 协议）
pip install fp-agent fp-webui fp-acp
```

`fp-agent` 会自动拉取 `fp-core`（核心引擎）和 `fp-terminal`（终端 REPL）两个依赖子包。如需 Web 界面或远程调用能力，额外安装 `fp-webui` 和 `fp-acp`。

| 包名 | PyPI 项目 | 安装命令 | 功能 |
|------|-----------|---------|------|
| 🔧 核心引擎 | `fp-core` | 自动依赖 | Agent 主干、LLM 通信、工具执行器、会话管理 |
| 🖥️ 终端 REPL | `fp-terminal` | 自动依赖 | 交互式 CLI、斜杠命令、Tab 补全、Markdown 渲染 |
| 🌐 Web 界面 | `fp-webui` | `pip install fp-webui` | FastAPI + 浏览器图形界面 |
| 🔗 ACP 协议 | `fp-acp` | `pip install fp-acp` | JSON-RPC 2.0 远程调用，IDE 集成 |
| 🎯 主入口 | `fp-agent` | `pip install fp-agent` | 统一路由入口，自动派发到各子界面 |

### 环境要求

| 项目 | 要求 |
|------|------|
| Python | ≥ 3.11 |
| LLM API Key | 兼容 OpenAI 接口的 API 密钥（DeepSeek / OpenAI / 通义千问 等） |
| 操作系统 | Linux / macOS / Windows |
| Windows 推荐 | **Git for Windows**（可选，[git-scm.com](https://git-scm.com) 安装时勾选"Git Bash"并添加到 PATH）。安装后可获得完整 Unix 命令（`ls`/`grep`/`awk`/`sed`）；未安装时自动降级到 cmd.exe（仅支持 `dir`/`type`/`findstr` 等 Windows 原生命令） |

## ⚙️ 二、初始化

### 1. 配置 API Key

首次运行前，需要设置 LLM API 密钥。支持两种方式：

**方式一：环境变量（推荐）**

```bash
export LLM_API_KEY="sk-your-api-key-here"
```

`fp` 会依次检查以下环境变量：`LLM_API_KEY` → `OPENAI_API_KEY`。

**方式二：配置文件**

第一次运行时，`fp` 会自动生成默认配置文件 `~/.config/fp/config.json`，编辑填入密钥：

```json
{
  "LLM_API_KEY": "sk-your-api-key-here",
  "LLM_MODEL": "deepseek-v4-flash",
  "LLM_API_BASE_URL": "https://api.deepseek.com/v1"
}
```

> **配置优先级**：配置文件显式值 > 环境变量 > 代码默认值。
> 完整配置项说明见 [配置指南](配置指南.md)。

### 2. 首次启动

```bash
fp
```

启动成功后会显示 ASCII Logo 和提示信息：

```
╭──────────────────────────────────────────╮
│    Five Pebbles Agent  0.1.2          │
│  输入 /help 查看命令列表                   │
│  Ctrl+C 中断  |  Ctrl+D 保存退出           │
╰──────────────────────────────────────────╯
```

此时你就进入了交互式 REPL，可以直接输入消息与 AI 对话。

### 3. 命令行参数

```bash
fp                           # 交互模式（默认）
fp --mode webui              # 启动 Web 界面（需安装 fp-webui）
fp --mode acp                # 启动 ACP 协议服务（需安装 fp-acp）
fp --model gpt-4o            # 指定模型（覆盖配置文件）
fp --session abc123          # 恢复指定会话
```

> 详细用法参见 [CLI 入门](CLI入门.md)。

## 🧩 三、内置系统

Five Pebbles 有三套核心内置系统——**技能**（Skills）、**工具**（Tools）和**命令**（Commands）。它们协同工作：技能定义了 AI 知道什么，工具定义了 AI 能做什么，命令定义了你怎么控制 AI。

### 🔧 工具系统（Tools）

工具是 Agent 的执行手臂——每个工具是一个可被 LLM 调用的函数，执行具体操作（运行命令、读写文件、搜索网络等）。

**核心工具**（4 个，始终可用）：

| 工具 | 功能 |
|------|------|
| `bash` | 执行任意 shell 命令，支持管道/重定向 |
| `read_file` | 读取文件内容，支持行范围 |
| `write_file` | 创建或覆写文件 |
| `edit_file` | 精确字符串替换修改文件 |

**插件工具**（11 个，通过技能系统动态注册）：

| 工具 | 功能 |
|------|------|
| `web_search` | 快速搜索，返回标题/链接/摘要 |
| `web_fetch` | 抓取网页内容 |
| `python` | 执行 Python 代码（数据处理/算法验证） |
| `memory_read` | 读取跨会话持久化记忆 |
| `memory_save` | 保存跨会话记忆 |
| `subagent` | 派遣子 Agent 独立执行任务 |
| `task_create` / `task_update` / `task_list` / `task_clear` | 多步骤任务追踪 |
| `file_fingerprint` | 文件指纹识别 |
| `kdeconnect_notify` | 手机通知推送 |
| `vision` | 图像识别（调用视觉模型） |
| 更多... | 参见 [工具概览](工具概览.md) 完整列表 |

> **工具调用流程**：用户输入 → LLM 分析意图 → LLM 决定调用哪个工具及参数 → 系统执行 → 结果返回 LLM → LLM 生成最终回复。整个过程在终端以 `🛠️` 图标实时显示。

### 🎯 技能系统（Skills）

技能是 Agent 的能力定义——每个技能是一个 `.md` 文件，包含 YAML frontmatter 和 Markdown 描述。LLM 读取这些文件后，就知道在什么场景下该使用什么工具。

**内置技能（13 个）：**

| 分类 | 技能 | 功能 |
|------|------|------|
| 🔍 分析 | `file_fingerprint` | 文件类型识别（file + strings） |
| | `process_tracking` | 进程行为追踪（系统调用/网络活动） |
| | `text_processing` | 日志/配置文件模式提取 |
| | `elf_analysis` | ELF 可执行文件结构分析 |
| | `privilege_escalation_scan` | SUID/SGID 权限提升扫描 |
| | `env_pollution_check` | 环境变量污染检测 |
| 📥 下载 | `yt_dlp_download` | YouTube 等网站音视频下载 |
| 💻 代码 | `python_syntax_check` | Python 语法检查 |
| | `lanqiao_fetch` | 蓝桥杯题目抓取 |
| | `lanqiao_submit` | 蓝桥杯代码提交判题 |
| ⚙️ 系统 | `modify_skills` | 添加/修改/删除技能文件 |
| | `delete_empty_sessions` | 清理空会话文件 |
| | `self_modification` | 修改自身源代码并测试 |

**技能管理：**

```bash
/skills                   # 列出所有已加载技能
/reload_skills            # 热重载技能（修改 .md 后无需重启）
/remove_skill <name>      # 运行时移除指定技能
```

> 技能文件存储在 `~/.local/share/fp/skills/` 目录下，AI 可通过工具自主增删改技能文件。详情见[技能系统](技能系统.md)。

### 🎮 命令系统（Commands）

命令是**你**控制 Agent 的方式——所有以 `/` 开头的斜杠命令，用于管理会话、技能、记忆和系统行为。

**会话管理：**

| 命令 | 功能 |
|------|------|
| `/help` (`/?`) | 显示所有可用命令 |
| `/exit` (`/quit`) | 正常退出，显示统计面板 |
| `/exit!` | 核弹退出：删除当前会话不留痕迹 |
| `/clear` | 清空当前会话历史 |
| `/reset` | 重置上下文到初始状态 |
| `/back` | 回退到历史某处 |
| `/fork` | 基于当前上下文创建新会话 |
| `/compact` | 压缩对话历史节省 Token |
| `/resume` | 切换/删除历史会话 |

**系统管理：**

| 命令 | 功能 |
|------|------|
| `/skills` | 列出已加载技能 |
| `/reload_skills` | 热重载技能文件 |
| `/remove_skill <n>` | 移除指定技能 |
| `/memory` | 列出持久化记忆 |
| `/model` | 显示当前模型配置 |
| `/session` | 显示当前会话 ID |

> 所有命令支持 Tab 补全。完整参考见[命令参考](命令参考.md)。

## 🔌 四、外置系统：插件

如果说技能、工具、命令是内置的"硬连线"能力，插件就是 **扩展系统行为** 的软接口。插件通过挂载到 **生命周期钩子** 上，在系统运行的关键时刻自动执行自定义逻辑。

### 生命周期钩子

系统定义了 19 个钩子，覆盖从启动到关闭的完整流程：

| 阶段 | 钩子 | 触发时机 |
|------|------|---------|
| 🚀 **初始化** | `on_start` | 系统启动时 |
| | `on_ready` | 初始化完成时 |
| | `on_config_loaded` | 配置加载完成时 |
| 💬 **消息处理** | `on_user_message` | 收到用户消息时 |
| | `on_before_process` | 开始处理前 |
| | `on_after_process` | 处理完成后 |
| 🧠 **思考** | `on_before_think` / `on_after_think` | LLM 思考前后 |
| 📡 **LLM 调用** | `on_before_llm_call` / `on_after_llm_call` | LLM API 调用前后 |
| | `on_llm_error` | API 调用出错时 |
| 🛠️ **工具** | `on_before_tool_call` / `on_after_tool_call` | 工具调用前后 |
| 📤 **响应** | `on_before_response` / `on_after_response` | 回复生成前后 |
| 📦 **上下文** | `on_context_loaded` / `on_context_compacted` | 上下文加载/压缩时 |
| ⚠️ **错误** | `on_error` | 未捕获异常时 |
| 🚪 **资源** | `on_shutdown` | 系统关闭时 |

### 内置插件

**通知插件**：在 Agent 回复完成时（`on_after_response`）推送桌面通知；发生错误时（`on_error`/`on_llm_error`）发出告警。

### 插件开发

插件系统的核心理念是 **"文件即开关"**——将 `.py` 文件放入插件目录即加载，移出即卸载，无需任何配置变更。

```python
from .base import Plugin

class MyPlugin(Plugin):
    """一个简单的示例插件"""

    def on_register(self):
        self.register_hook("on_user_message", self.on_user_message)

    async def on_user_message(self, message: str):
        if "hello" in message.lower():
            print("🌍 用户说了 hello！")
```

> 完整开发指南见[插件系统](插件系统.md)和[开发者文档](../dev/插件系统.md)。
