## 1. 简介

**Five Pebbles（五块卵石）** 是一个基于大型语言模型的 AI Agent（智能体）程序。它不是一个简单的聊天机器人，而是一个拥有**完整工具链**、**技能系统**、**会话管理**、**记忆系统**和**插件机制**的智能体框架。

### 核心特性

| 特性 | 说明 |
|------|------|
| 🧠 **LLM 驱动** | 接入任意 OpenAI 兼容 API（DeepSeek、OpenAI、Claude 等） |
| 🔧 **工具调用** | 可执行 Shell 命令、读写文件、编辑文件、搜索互联网、执行 Python 代码 |
| 📂 **会话管理** | 每次对话自动保存为 JSONL 文件，支持恢复、分支（fork） |
| 📚 **技能系统** | 通过 Markdown 文件热加载技能，让 Agent 学会新能力 |
| 🧩 **插件系统** | 生命周期钩子驱动的插件架构，可扩展桌面通知等 |
| 💾 **记忆系统** | 文件型持久化记忆，跨会话保留关键信息 |
| 🎨 **精美终端 UI** | ANSI 配色、Rich Markdown 渲染、启动 Logo、关闭面板统计 |
| ⌨️ **智能输入** | Tab 补全、命令历史、斜杠命令系统 |

## 2. 快速开始

### 2.1 环境要求

- Python 3.9+
- pip 环境
- 一个兼容 OpenAI 接口的 LLM API Key

### 2.2 安装

```bash
# 克隆或进入项目目录
cd /media/zpb/data/codes/AI/agent

# 安装依赖（建议使用虚拟环境）
pip install -r requirements.txt
```

核心依赖：
- `httpx` — 异步 HTTP 客户端（LLM API 通信）
- `prompt_toolkit` — 交互式输入（Tab 补全、历史记录）
- `rich` — Markdown 渲染（可选，LLM 回复美化）
- `wcwidth` — 终端字符宽度计算（可选，界面对齐）

### 2.3 配置 API Key

**方式一：编辑 `config.json`**

```bash
python3 cli.py --init
```

这会生成默认的 `config.json`，编辑其中的 `LLM_API_KEY` 字段：

```json
{
  "LLM_API_KEY": "sk-your-api-key-here",
  "LLM_API_BASE_URL": "",
  "LLM_MODEL": ""
}
```

**方式二：设置环境变量**

```bash
export LLM_API_KEY="sk-your-api-key-here"
export LLM_API_BASE_URL=""
export LLM_MODEL=""
```

> **优先级**：`config.json` > 环境变量 > 代码默认值

### 2.4 首次启动

```bash
python3 cli.py
```

启动后你会看到 ASCII Logo 和启动信息：

```
🤖 Five Pebbles 已启动 (模型: deepseek-chat)

💡 输入 /help 查看命令，/resume 可回到历史会话

(Agent) > _
```

现在你可以直接输入问题，Five Pebbles 会调用各种工具来帮你解决问题。

## 3. 配置说明

### 3.1 `config.json` 完整字段

配置文件位于项目根目录，JSON 格式：

```json
{
  "LLM_API_KEY": "sk-xxx",
  "LLM_API_BASE_URL": "",
  "LLM_MODEL": "",
  "TEMPERATURE": 0.8,
  "MAX_TOKENS": 32768,
  "TIMEOUT": 300,
  "RETRY_COUNT": 3,
  "MAX_ITERATIONS": 50,
  "MAX_CONTEXT_TOKENS": 8000,
  "MEMORY_MAX_HISTORY": 100,
  "display_styles": {
    "info": {"color": "green"},
    "error": {"color": "bright_red", "bold": true},
    "warning": {"color": "yellow"},
    "hint": {"color": "cyan"},
    "llm_thought": {"color": "magenta", "dim": true},
    "llm_tool": {"color": "yellow"},
    "startup": {"color": "cyan", "bold": true},
    "logo": {"color": "cyan", "bold": true}
  },
  "display_truncation": {
    "llm_thought": 100,
    "llm_tool": 300,
    "logo": -1
  }
}
```

### 3.2 字段详解

#### LLM 连接配置

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `LLM_API_KEY` | string | `""` | **必填**。API 密钥。可设为环境变量同名。 |
| `LLM_API_BASE_URL` | string | `"https://api.deepseek.com/v1"` | API 基础地址。兼容 OpenAI 格式的任意服务商。 |
| `LLM_MODEL` | string | `"deepseek-v4-flash"` | 模型名称。 |
| `TEMPERATURE` | float | `0.8` | 生成温度，0~2。越高越随机，越低越确定。 |
| `MAX_TOKENS` | int | `32768` | 单次 LLM 调用最大输出 Token 数。 |
| `TIMEOUT` | int | `300` | API 请求超时时间（秒）。 |
| `RETRY_COUNT` | int | `3` | API 请求失败重试次数。 |

#### Agent 行为配置

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `MAX_ITERATIONS` | int | `50` | 单次交互的最大思考-工具调用的循环次数。防止死循环。 |
| `MAX_CONTEXT_TOKENS` | int | `8000` | 上下文窗口的 token 估算上限。超出时自动裁剪早期消息。 |
| `MEMORY_MAX_HISTORY` | int | `100` | 记忆历史保留条数。 |

#### 显示配置

**`display_styles`**：按名称配置各 UI 元素的 ANSI 颜色和样式。

可用颜色：`black`, `red`, `green`, `yellow`, `blue`, `magenta`, `cyan`, `white`, `bright_red`, `bright_green`, `bright_yellow`, `bright_cyan`, `bright_white`, `default`

可用样式修饰：`bold`（粗体）、`dim`（暗淡）、`italic`（斜体）

样式名称对照表：

| 名称 | 用途 |
|------|------|
| `info` | 操作反馈信息（绿色） |
| `item` | 列表子条目（默认色） |
| `hint` | 行为提示/引导（青色） |
| `error` | 错误信息（亮红+粗体） |
| `warning` | 警告信息（黄色） |
| `llm_thought` | LLM 思考过程（品红+暗淡） |
| `llm_tool` | 工具调用信息（黄色） |
| `llm_output` | LLM 回复内容（默认色） |
| `debug` | 调试日志（品红+暗淡） |
| `startup` | 启动横幅（青色+粗体） |
| `logo` | ASCII Logo 区域（青色+粗体） |

**`display_truncation`**：按名称配置显示截断长度。

| 名称 | 默认值 | 说明 |
|------|--------|------|
| `llm_thought` | `100` | LLM 思考内容截断字符数 |
| `llm_tool` | `300` | 工具调用信息截断字符数 |
| `logo` | `-1` | Logo 截断（-1=不截断） |

> 截断时末尾会追加 `… <+N chars>` 标记剩余字符数。

## 4. CLI 使用

### 4.1 基本用法

```bash
python3 cli.py
```

进入交互模式，可连续对话。

### 4.2 命令行参数

| 参数 | 说明 |
|------|------|
| `-m`, `--message TEXT` | **单次消息模式**。发送一条消息后立即退出。 |
| `-r`, `--resume [SESSION_ID]` | 恢复历史会话。不传参数时自动恢复最新会话。 |
| `--init` | 初始化配置文件（生成默认 `config.json`）。 |

### 4.3 示例

```bash
# 交互模式
python3 cli.py

# 单次消息模式
python3 cli.py -m "列出当前目录的文件"

# 恢复最新会话
python3 cli.py -r

# 恢复指定会话
python3 cli.py -r s_260606_160052583847

# 初始化配置
python3 cli.py --init
```

## 5. 所有斜杠命令

Five Pebbles 支持以 `/` 开头的斜杠命令。输入 `/help` 可随时查看命令列表。

### 5.1 命令速查表

| 命令 | 别名 | 说明 |
|------|------|------|
| `/help` | `/?` | 显示帮助信息 |
| `/exit` | `/quit` | 正常退出程序（会生成会话总结） |
| `/exit!` | — | **核弹级退出**：删除当前会话文件，不留痕迹 |
| `/clear` | — | 清空当前会话历史 |
| `/reset` | — | 重置上下文（保留系统提示词和技能） |
| `/back` | — | 回退到对话的某个历史时刻 |
| `/history` | — | 查看当前对话历史 |
| `/fork` | — | 基于当前上下文新建分支会话 |
| `/compact` | — | 压缩对话历史（用 LLM 将早期消息提炼为摘要） |
| `/resume` | — | 切换历史会话 |
| `/reload_skills` | — | 热重载技能文件 |
| `/remove_skill` | — | 删除指定技能 |
| `/skills` | — | 列出所有已加载的技能 |
| `/model` | — | 显示当前模型配置信息 |
| `/memory` | — | 查看持久化记忆列表 |
| `/session` | — | 显示当前会话 ID |

### 5.2 命令详解

#### `/help` 或 `/?`

显示所有可用命令及其简短描述。

```
(Agent) > /help
```

#### `/exit` 或 `/quit`

正常退出程序。退出时 LLM 会自动为会话生成总结（5~10 个汉字的标题），并显示关闭面板统计（消息数、耗时、模型等）。会话文件会保存。

```
(Agent) > /exit
```

退出面板示例：

```
╔════════════════════════════════════════════════════╗
║  📂  会话结束                                       ║
║                                                    ║
║  总结: 代码调试与优化                                ║
║  文件: s_260606_160052583847.jsonl                  ║
║                                                    ║
║  📊  统计信息                                       ║
║                                                    ║
║  模型: deepseek-v4-flash                            ║
║  消息: 23 条                                       ║
║  创建: 2026-06-06 16:00:52                         ║
║  耗时: 1:23:45                                     ║
╚════════════════════════════════════════════════════╝

👋  再见！
```

#### `/exit!`

**核弹级退出**。退出前删除当前会话的 JSONL 文件，不生成总结、不留任何痕迹。

```
(Agent) > /exit!
💥 会话 s_260606_160052583847 已删除，不留痕迹
```

> ⚠️ 此操作不可逆，请谨慎使用。

#### `/clear`

清空当前会话的所有历史消息，但**保留**系统提示词和当前会话 ID。相当于新建一个干净的对话。

```
(Agent) > /clear
🧹 当前会话已清空
```

#### `/reset`

重置上下文到初始状态。与 `/clear` 不同的是，它会重新加载系统提示词和技能，恢复到刚启动时的状态。

```
(Agent) > /reset
🔄 上下文已重置
```

#### `/back`

回退到对话的某个历史时刻。执行后会显示所有历史消息（带编号），输入编号即可回退。

```
(Agent) > /back

📜 对话历史（共 12 条消息）:

  [  1] 👤 用户: 帮我分析一下这个日志文件
  [  2] 🤖 AI: 好的，让我先看看文件内容...
  [  3] 🔧 工具: 调用了 bash...
  ...
  [ 12] 🤖 AI: 结论是数据库连接超时

  输入 1~12 回退到对应位置
  输入 q 取消

AI <- 选择: 6
⏪ 已回退到第 6 条消息位置
```

#### `/history`

查看当前会话的所有历史消息摘要。每条消息显示角色、前 120 个字符的内容。

```
(Agent) > /history

📜 对话历史（共 8 条）:

  [  1] 👤 用户: 我的服务器日志在哪里？
  [  2] 🤖 AI: 让我检查一下常用的日志目录...
  ...
```

#### `/fork`

基于当前会话的完整上下文**创建新会话**，然后切换到新会话。原会话会被保存（带摘要标签）。

适用于：想保留当前讨论的所有上下文，但在新会话中继续探索不同方向。

```
(Agent) > /fork
🍴 已 fork：从 s_260606_160052583847 → s_260606_163012345678
```

#### `/compact`

当对话历史过长时，用 LLM 将早期消息（最近 4 条之前的消息）压缩为一段摘要，节省上下文空间。

```
(Agent) > /compact
🔄 正在压缩对话历史...
✅
📦 已压缩 15 条消息为摘要
```

压缩后的上下文结构：
1. 系统提示词（含技能）
2. 压缩摘要（system 角色）
3. 最近 4 条消息（保留完整内容）

#### `/resume`

切换历史会话。三种用法：

```bash
# 交互式选择（列出所有会话，输入编号切换）
/resume

# 切换到最新会话
/resume latest

# 切换到指定会话 ID
/resume s_260606_160052583847
```

交互模式示例：

```
(Agent) > /resume

📂 历史会话列表（输入编号切换，q 取消）:
  [1] 代码调试与优化                   (23条, 2026-06-06 16:00:52)  ⬅
  [2] API 接口测试                     ( 8条, 2026-06-05 14:22:10)
  [3] 系统性能分析                     (15条, 2026-06-04 09:15:33)

请输入编号 (1-3, 或 q 取消): 2
📂 已切换到会话
```

#### `/reload_skills`

热重载 `skills/` 目录下的所有技能文件。修改技能文件后无需重启程序即可生效。

```
(Agent) > /reload_skills
🔄 正在重新加载技能...
✅ 技能重载完成！当前可用技能数：13
  • 文件指纹识别 (file_fingerprint)
  • 进程行为动态追踪 (process_tracking)
  • 高级文本处理 (text_processing)
  ...
```

#### `/remove_skill <技能名>`

从运行中的 Agent 移除指定技能（不删除文件）。

```
(Agent) > /remove_skill file_manager
✅ 技能已删除：file_manager
🔄 系统提示词已更新，技能已移除！
```

不传参数时会显示当前可用技能列表。

#### `/skills`

列出当前所有已加载的技能及其描述。

```
(Agent) > /skills
📚 当前可用技能:

  • 文件指纹识别 (file_fingerprint)

  • 进程行为动态追踪 (process_tracking)
    实时监控系统调用与网络活动，定位可疑进程。
  ...
```

#### `/model`

显示当前 LLM 配置和重要路径信息。

```
(Agent) > /model
模型:   deepseek-v4-flash

最大 Token: 32768
```

#### `/memory`

列出所有已保存的持久化记忆文件。

```

📋 记忆列表:

  • project_goal

💡 使用 memory_read / memory_save 工具管理记忆
```

#### `/session`

显示当前会话的 ID。

```
(Agent) > /session
📂 当前会话: s_260606_160052583847
```

## 6. 技能系统

技能系统是 Five Pebbles 的**核心扩展机制**。每个技能是一个 Markdown 文件（带 YAML frontmatter），包含技能名称、描述和详细的命令示例。Agent 将技能内容作为系统提示词的一部分，使其"学会"这些能力。

### 6.1 技能文件位置

所有技能文件位于项目根目录的 `skills/` 文件夹下：

```
skills/
├── file_fingerprint.md
├── process_tracking.md
├── text_processing.md
├── elf_analysis.md
├── privilege_escalation_scan.md
├── env_pollution_check.md
├── yt_dlp_download.md
├── python_syntax_check.md
├── lanqiao_fetch.md
├── lanqiao_submit.md
├── modify_skills.md
├── delete_empty_sessions.md
├── self_modification.md
└── loader.py          # 技能加载器（代码文件，非技能）
```

### 6.2 技能文件格式

每个技能文件使用 **YAML frontmatter** + **Markdown 正文** 的格式：

```markdown
---
name: file_fingerprint           # 技能标识名（唯一）
title: 文件指纹识别               # 技能显示标题
description: 使用 file 和 strings 命令提取特征，判断类型。  # 技能描述
category: security               # 分类（可选）
version: 1.0                     # 版本号（可选）
priority: 5                      # 优先级 1~10（可选，越高越靠前）
---

[技能] 文件指纹识别
当遇到未知二进制文件时，使用 `file` 和 `strings` 命令提取特征，判断类型。

命令示例：
- file <文件名>          # 判断文件类型（如：ELF, PDF, ZIP）
- strings <文件名> | head -20  # 显示可读字符串
```

**frontmatter 字段说明：**

| 字段 | 必须 | 类型 | 说明 |
|------|------|------|------|
| `name` | ✅ | string | 技能标识名，用于引用和删除 |
| `title` | ✅ | string | 技能显示标题 |
| `description` | ✅ | string | 简述技能用途 |
| `category` | ❌ | string | 分类标签（如 security, devops, algorithm） |
| `version` | ❌ | string | 版本号 |
| `priority` | ❌ | int | 在提示词中的排序优先级（1~10，默认 5） |

### 6.3 管理技能

**列出技能**：`/skills`

**热重载技能**：修改 `skills/` 下的 `.md` 文件后，执行 `/reload_skills` 即可生效，无需重启。

**动态添加技能**：将 `.md` 文件放入 `skills/` 目录，执行 `/reload_skills`。或使用 `/add_skill <文件名>`（在交互式 CLI 中暂不公开，但代码支持）。

**删除技能**：`/remove_skill <name>` 从运行中的 Agent 移除（不删除文件）。要从磁盘彻底删除，直接删除 `skills/` 下的对应 `.md` 文件。

### 6.4 内置技能一览

| 技能文件 | 名称 | 说明 |
|----------|------|------|
| `file_fingerprint.md` | 文件指纹识别 | 用 `file`、`strings` 分析文件类型 |
| `process_tracking.md` | 进程行为动态追踪 | 监控系统调用和网络活动 |
| `text_processing.md` | 高级文本处理 | 用 `awk`/`sed`/`rg` 处理日志和文本 |
| `elf_analysis.md` | ELF 逆向分析 | 用 `readelf`/`objdump` 分析二进制文件 |
| `privilege_escalation_scan.md` | 权限提升扫描 | 检测 SUID/SGID 文件和写权限 |
| `env_pollution_check.md` | 环境变量污染检测 | 检查 PATH/LD_LIBRARY_PATH 安全性 |
| `yt_dlp_download.md` | 视频下载 | 用 yt-dlp 下载音视频 |
| `python_syntax_check.md` | Python 语法检查 | 用 `ast` 模块检查语法 |
| `lanqiao_fetch.md` | 蓝桥杯题目抓取 | 从蓝桥云课获取题目信息 |
| `lanqiao_submit.md` | 蓝桥杯代码提交判题 | 提交代码到 OJ 判题系统 |
| `modify_skills.md` | 添加/修改/删除技能 | 管理技能文件 |
| `delete_empty_sessions.md` | 清理空会话 | 删除大小为 0 的会话文件 |
| `self_modification.md` | 修改自身源代码 | Agent 修改自己的代码并测试验证 |

## 7. 快捷键

| 快捷键 | 功能 |
|--------|------|
| **Tab** | 补全建议。输入 `/` 开头时自动补全命令名。始终确认当前选中项。 |
| **Ctrl+C** | 中断当前操作。在 LLM 思考/工具执行时按可中断并返回到输入提示符。 |
| **Ctrl+D** | 退出程序。在空输入行按下等同于 `/exit`。 |
| **↑/↓** | 浏览历史输入（prompt_toolkit 内置） |
| **Meta+Enter** | 打开外部编辑器编写多行输入（prompt_toolkit 支持） |

### 7.1 Tab 补全详解

输入 `/` 后按 Tab，会弹出补全菜单，显示所有可用命令。每个命令旁附带简短描述，帮助你快速选择。

```
(Agent) /h<Tab>
  /help        显示此帮助
  /history     查看当前对话历史
  /compact     压缩对话历史
```

Tab 键的行为是**确认当前选中项**（而非循环选择），这更符合直觉。

### 7.2 Ctrl+C 中断处理

Five Pebbles 采用**双保险机制**处理 Ctrl+C 中断：

1. **Unix 主线程**：直接取消所有 asyncio 任务（注入 `CancelledError`）
2. **跨平台回退**：设置全局 `_interrupted_flag`，在下一次检查点响应

中断后的行为：
- **LLM 思考中中断**：保留已生成的内容，标记中断
- **工具执行中中断**：当前工具标记为"调用失败"，剩余工具标记为"未执行"，退出循环

## 8. 会话管理

### 8.1 自动保存

每次交互后，Agent 自动将会话保存到 `data/sessions/` 目录。每个会话用一个 JSONL 文件存储。

**文件命名格式**：`s_YYMMDD_HHMMSSffffff.jsonl`

**文件结构**：

```
第 1 行: {"__meta__": true, "id": "s_260606_160052583847", "created": "2026-06-06 16:00:52", "updated": "2026-06-06 17:24:37", "summary": "代码调试与优化", "message_count": 23}
第 2 行: {"role": "user", "content": "帮我分析日志..."}
第 3 行: {"role": "assistant", "content": "好的，让我先看看...", "tool_calls": [...]}
...
```

> 第 1 行是嵌入的元数据（meta），包含会话 ID、创建/更新时间、摘要、消息数。
> 后续每行一条消息，按时间**正序存储**，最新消息在末尾。

### 8.2 会话列表

`/resume` 命令（无参数）会列出所有历史会话：

```
(Agent) > /resume

📂 历史会话列表（输入编号切换，q 取消）:
  [1] 代码调试与优化                   (23条, 2026-06-06 16:00:52)  ⬅
  [2] API 接口测试                     ( 8条, 2026-06-05 14:22:10)
  [3] 系统性能分析                     (15条, 2026-06-04 09:15:33)

请输入编号 (1-3, 或 q 取消):
```

### 8.3 恢复会话

```bash
# CLI 启动时恢复最新会话
python3 cli.py -r

# CLI 启动时恢复指定会话
python3 cli.py -r s_260606_160052583847

# 运行中切换到最新会话
/resume latest

# 运行中交互式选择
/resume
```

### 8.4 分支（Fork）

`/fork` 命令基于当前会话的所有上下文创建**新会话**。原会话会被保存，并自动生成摘要。新会话包含完全相同的历史消息，但有了新的会话 ID。

适用场景：
- 想尝试不同的解决路径
- 在关键决策点保存一个"检查点"
- 从长对话中分支出一个子话题

### 8.5 会话文件管理

会话文件路径：`data/sessions/s_YYMMDD_HHMMSSffffff.jsonl`

你可以直接操作这些 JSONL 文件：
- **备份**：复制文件到其他地方
- **删除**：删除文件释放空间（下次启动时不会显示该会话）
- **查看**：`cat data/sessions/s_*.jsonl` 查看原始内容

如需清理空会话文件（大小为 0），Agent 内置了 `delete_empty_sessions` 技能，会自动扫描并清理。

## 9. 记忆系统

### 9.1 什么是记忆

记忆系统允许 Agent 在**跨会话**之间保留关键信息。与"会话历史"不同，记忆是持久化的——即使你启动了新会话，记忆仍然可用。

每个记忆存储为一个带 YAML frontmatter 的 Markdown 文件：

```
data/memory/
├── project_goal.md
├── user_role.md
├── ...
```

### 9.2 记忆格式

```markdown
---
name: project_goal
description: AI Agent 项目目标
type: project
created: 2026-06-01 14:30:00
---

Five Pebbles Agent v2 项目目标：
- 创建一个完整的 AI Agent 框架
- 支持工具调用、技能系统、会话管理
- 插件化的生命周期架构
```

### 9.3 管理记忆

Agent 在对话中通过两个工具管理记忆：

**`memory_save`** — 保存新记忆：

```python
memory_save(name="project_goal", type="project",
            description="AI Agent 项目目标",
            content="Five Pebbles Agent v2 项目目标：...")
```

**`memory_read`** — 读取记忆（不传参数列出全部，传关键词搜索）：

```python
memory_read(query="project")        # 搜索包含 project 的记忆
memory_read()                        # 列出所有记忆
```

**`/memory`** 命令查看记忆列表：

```
(Agent) > /memory
📋 记忆列表:

  • project_goal
  • user_role

💡 使用 memory_read / memory_save 工具管理记忆
```

### 9.4 记忆的类型

| 类型 | 说明 |
|------|------|
| `user` | 用户个人信息/偏好 |
| `project` | 项目状态/关键决策 |
| `feedback` | 用户的反馈和偏好 |
| `reference` | 参考资料/知识库 |

## 11. 插件系统

### 11.1 架构

Five Pebbles 采用**生命周期钩子**驱动的插件架构。共定义了 19 个生命周期钩子，覆盖 Agent 运行的各个阶段：

| 阶段 | 钩子 |
|------|------|
| 🚀 **初始化** | `ON_INIT`, `ON_CONFIG_LOADED` |
| 💬 **消息处理** | `ON_MESSAGE_RECEIVED`, `ON_MESSAGE_PARSE`, `ON_MESSAGE_FILTER` |
| 🤔 **思考** | `ON_BEFORE_THINK`, `ON_THINK`, `ON_AFTER_THINK` |
| 📡 **LLM 调用** | `ON_BEFORE_LLM_CALL`, `ON_LLM_CALL`, `ON_AFTER_LLM_CALL` |
| 📤 **响应** | `ON_BEFORE_RESPONSE`, `ON_RESPONSE`, `ON_AFTER_RESPONSE` |
| 🔧 **工具** | `ON_TOOL_SELECT`, `ON_TOOL_CALL`, `ON_TOOL_RESULT`, `ON_TOOL_ERROR` |
| 📦 **上下文** | `ON_CONTEXT_UPDATE`, `ON_CONTEXT_READ` |
| ⚠️ **错误** | `ON_ERROR`, `ON_RETRY` |
| 🧹 **资源** | `ON_SHUTDOWN`, `ON_CLEANUP` |

### 11.2 内置插件

**通知插件**（`plugins/notification.py`）：

监听关键生命周期事件，提供：
- **桌面通知**：通过 `notify-send` 发送（支持 Linux）
- **声音提醒**：通过 `paplay` 播放系统提示音

通知事件：错误发生、回复完成、程序关闭

配置示例（在代码中注册）：

```python
from plugins.notification import create_notification_plugin

# 启用桌面通知和声音
agent.plugins.register(create_notification_plugin(
    desktop=True,
    sound=True,
    rate_limit=2.0,       # 通知频率限制（秒）
    max_desktop_len=120,  # 通知正文最大长度
))
```

### 11.3 插件开发

插件需继承 `Plugin` 基类，实现 `on_register()` 方法：

```python
from plugins.base.plugin import Plugin, PluginConfig
from core.lifecycle import LifecycleHook, HookContext

class MyPlugin(Plugin):
    name = "my_plugin"
    version = "1.0.0"

    def on_register(self, lifecycle):
        lifecycle.register(
            LifecycleHook.ON_MESSAGE_RECEIVED,
            self.on_message,
            priority=50,
            name="my_on_message"
        )

    async def on_message(self, ctx: HookContext, **kwargs):
        content = kwargs.get("content", "")
        print(f"[MyPlugin] 收到消息: {content[:50]}...")
        return ctx
```

## 12. 常见问题/故障排除

### Q1: 启动时提示 "LLM_API_KEY not set"

```
[Warning] LLM_API_KEY not set
  → 设置方式：① 修改 config.json ② 设置环境变量 OPENAI_API_KEY
```

**解决方法**：
1. 编辑 `config.json`，填入真实的 API Key
2. 或设置环境变量：`export LLM_API_KEY="sk-xxx"`
3. 也可设置 `export OPENAI_API_KEY="sk-xxx"`（环境变量名可不同，看代码配置）

### Q2: API 请求超时

```
错误：API/LLM 错误: timed out
```

**可能原因和解决方法**：
- 网络连接问题：检查能否访问 API 服务商
- 模型负载高：减少上下文长度或降低 `MAX_TOKENS`
- API Base URL 配置错误：确认 `LLM_API_BASE_URL` 正确
- 超时时间太短：增大 `TIMEOUT` 配置（默认 300 秒）

### Q3: Agent 陷入死循环，一直调工具

Five Pebbles 内置了两层防护：

1. **`MAX_ITERATIONS`**（默认 50）：单次交互最多循环 50 次，超限会自动退出并告知用户
2. **死循环检测器**：分析连续多次相同的工具调用模式

如果遇到循环问题：
```
⚠️  达到最大迭代次数 (50)，强制跳出循环
```

按 **Ctrl+C** 可随时中断当前操作。

### Q4: 上下文超长，Agent 忘记了早期内容

上下文窗口受 `MAX_CONTEXT_TOKENS`（默认 8000 tokens）限制。超限时自动裁剪最早的消息。

**优化建议**：
- 执行 `/compact` 命令压缩早期消息为摘要
- 增大 `MAX_CONTEXT_TOKENS` 配置（取决于 LLM 模型的上下文窗口大小）
- 使用 `/fork` 在关键点创建新会话，避免单会话过长

### Q5: Ctrl+C 中断后程序无响应

```
⏹️ 已中断（保留了已生成的内容）
```

如果中断后界面卡住，可能是因为：

1. 子进程尚未终止（如 `bash` 工具执行的耗时命令）
2. 中断标志残留

**解决方法**：
- 再次按 Ctrl+C
- 如果完全卡死，可以 `kill` 进程：`pkill -f cli.py`
- 会话文件已自动保存，下次可用 `/resume` 恢复

### Q6: 技能修改后未生效

确认是否执行了 `/reload_skills`。修改 `skills/*.md` 文件后，必须执行热重载命令才能生效。

```
(Agent) > /reload_skills
🔄 正在重新加载技能...
✅ 技能重载完成！当前可用技能数：13
```

### Q7: 会话文件损坏或无法恢复

JSONL 文件可能因异常写入而损坏。尝试：
1. 检查文件第一行是否为有效的 JSON meta
2. 手动编辑修复损坏的行
3. 使用 `/compact` 或启动新会话

### Q8: 提示 "文件不存在" 错误

确保文件路径是**绝对路径**。Agent 的当前工作目录是项目根目录，使用相对路径时可能不符合预期。

### Q9: Tab 补全没有反应

Tab 补全仅在输入以 `/` 开头时触发。在输入普通文本时按 Tab 不会有反应。

如果 `/` 开头也不补全：
- 确认已安装 `prompt_toolkit`：`pip install prompt_toolkit`
- 确认输入不是从文件重定向：补全仅支持 TTY 终端

### Q10: 如何迁移/备份？

核心数据目录：
```
data/
├── sessions/     # 会话文件（JSONL）
├── memory/       # 记忆文件（Markdown）
└── tasks.json    # 任务数据
```

备份这些目录即可完整迁移。恢复时放回原位置即可。
