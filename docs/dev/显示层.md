# 显示层

显示层负责所有面向用户（终端）的输出，涵盖 LLM 流式输出、日志、旋转动画和系统仪式感提示。

---

## 6 类输出函数

所有输出样式（颜色、标记）均从 `~/.config/fp/config.json` 的 `display_styles` 和 `display_truncation` 读取。

| 类别 | 函数 | 用途 |
|------|------|------|
| **A. 操作反馈** | `info(msg)` / `item(label, msg)` | 信息提示、分项展示 |
| **B. 行为提示** | `hint(msg)` | 引导用户下一步操作 |
| **C. 异常警示** | `error(msg)` / `warning(msg)` | 错误与警告输出 |
| **D. LLM 流** | `llm_thought(text)` / `llm_tool(text)` / `llm_output(text)` / `llm_newline()` / `llm_iteration(label)` | 流式渲染 LLM 思考、工具调用、最终回复 |
| **E. 系统日志** | `debug(msg)` | 调试级输出，受 config 中 debug 开关控制 |
| **🎨 仪式感** | `startup()` / `shutdown_panel()` / `print_logo()` | 启动/关闭时的品牌展示 |

实现位于 `fp_core/core/display.py`，所有函数最终调用底层 `_styled_write(style, text)`。

---

## LLMStreamer

`class LLMStreamer` 负责逐块接收 LLM 输出并渲染到终端。

```
class LLMStreamer:
    def think(self, text: str):     # 渲染思考片段
    def write(self, text: str):     # 渲染输出片段
    def end(self, interrupted=False): # 结束流（interrupted=True 表示被中断）
```

- 每次 `think()` / `write()` 调用立即刷新到 stdout。
- `end()` 负责清理尾部状态（如关闭 spinner、输出换行）。
- 中断模式下保留已输出内容，仅标记终止。

---

## Spinner

`class Spinner` 提供后台旋转动画，用于耗时操作的视觉反馈。

```
class Spinner:
    def start(self):   # 启动动画线程
    def stop(self):    # 停止并清理
```

- 动画帧序列从 `~/.config/fp/config.json` 的 `display_styles.spinner_frames` 读取。
- 所有终端输出前自动暂停 spinner，输出后恢复。

## 配置驱动样式

所有颜色/标记/截断长度均不硬编码：

```json
{
  "display_styles": {
    "info":     { "color": "cyan", "prefix": "ℹ" },
    "error":    { "color": "red",  "prefix": "✗" },
    "thought":  { "color": "green", "prefix": "◆" },
    "spinner_frames": ["⠋","⠙","⠹","⠸","⠼","⠴","⠦","⠧","⠇","⠏"]
  },
  "display_truncation": {
    "tool_output_max_len": 2000,
    "thought_max_len": 500
  }
}
```

代码中通过 `get_style(key)` 和 `get_truncation(key)` 访问。
