## 三级优先级

配置值按以下优先级顺序确定（从高到低）：

1. **`~/.config/fp/config.json`**（最高）— 用户 JSON 配置文件
2. **环境变量** — 系统环境变量（与配置键同名，如 `LLM_API_KEY`）
3. **代码默认值**（最低）— `_value()` 调用中传入的硬编码默认值

### `_value()` 实现逻辑

```python
def _value(key: str, default: Any = None) -> Any:
    """三优先级取值: JSON > 环境变量 > default"""
    # 1. JSON 中显式指定且不为 null
    if key in _json_cfg and _json_cfg[key] is not None:
        return _json_cfg[key]
    # 2. 环境变量（带类型转换）
    env_val = os.getenv(key)
    if env_val is not None:
        if key in ("TEMPERATURE",):
            return float(env_val)
        if key in ("MAX_TOKENS", "MAX_ITERATIONS", "TIMEOUT",
                    "RETRY_COUNT", "MAX_CONTEXT_TOKENS"):
            return int(env_val)
        return env_val
    # 3. 返回默认值
    return default
```

**类型转换**：环境变量值总是字符串，`_value()` 根据 key 自动转换：
- `TEMPERATURE` → `float`
- `MAX_TOKENS`, `MAX_ITERATIONS`, `TIMEOUT`, `RETRY_COUNT`, `MAX_CONTEXT_TOKENS` → `int`
- 其余保持 `str`

**注意**：配置键名使用**全大写 + 下划线**风格（如 `LLM_API_KEY`），与环境变量名一致。但部分 Python 模块变量加了 `LLM_` 前缀（如环境变量 `TEMPERATURE` → Python 变量 `LLM_TEMPERATURE`）。

## 配置 Schema 验证

### _CONFIG_SCHEMA

```python
_CONFIG_SCHEMA: dict[str, tuple[type, Any, Any, bool, str]] = {
    "LLM_API_KEY":      (str,   None, None,  True,  "API 密钥"),
    "LLM_API_BASE_URL": (str,   None, None,  True,  "API 基础地址"),
    "LLM_MODEL":        (str,   None, None,  True,  "模型名称"),
    "TEMPERATURE":      (float, 0.0,  2.0,   False, "生成温度"),
    "MAX_TOKENS":       (int,   1,    131072, False, "最大输出 token 数"),
    "TIMEOUT":          (int,   5,    600,    False, "请求超时秒数"),
    "RETRY_COUNT":      (int,   0,    10,     False, "失败重试次数"),
    "MAX_ITERATIONS":   (int,   1,    500,    False, "最大迭代轮次"),
    "MAX_CONTEXT_TOKENS": (int, 256,  131072, False, "上下文窗口 token 数"),
    "MEMORY_MAX_HISTORY": (int, 0,    10_000, False, "记忆保留历史条数"),
}
```

每条记录格式：`(期望类型, 最小值, 最大值, 是否必填, 描述)`

### _validate_value — 单键验证

```python
def _validate_value(key: str, value: Any, verbose: bool = False) -> list[str]:
```

验证流程：
1. **必填性检查** — 必填项为 `None` 或空字符串时报错
2. **类型检查** — `int` 可隐式兼容 `float`（如 JSON 中写 `TEMPERATURE: 1` 不报错）
3. **数值范围检查** — 超出 `[lo, hi]` 范围时报错

### validate_config — 全量验证

```python
def validate_config(verbose: bool = False) -> list[str]:
```

遍历 `_CONFIG_SCHEMA` 中所有键，从 `_json_cfg` 取值逐一验证，返回所有问题的汇总列表。

**模块加载时自动执行**：

```python
_validation_issues = validate_config()
if _validation_issues:
    for msg in _validation_issues:
        print(f"[Config] {msg}", file=sys.stderr)
```

因此启动时即可看到配置问题警告。

## 配置项一览表

| 环境变量 / JSON 键 | Python 变量 | 类型 | 默认值 | 说明 |
|-------------------|------------|------|--------|------|
| `LLM_API_KEY` | `LLM_API_KEY` | str | `""` | API 密钥（必填，未设置时 `check_llm_config()` 会告警） |
| `LLM_API_BASE_URL` | `LLM_API_BASE_URL` | str | `"https://api.deepseek.com/v1"` | API 基础地址（必填） |
| `LLM_MODEL` | `LLM_MODEL` | str | `"deepseek-v4-flash"` | 模型名称（必填） |
| `TEMPERATURE` | `LLM_TEMPERATURE` | float | `0.8` | LLM 采样温度（范围 0.0–2.0） |
| `MAX_TOKENS` | `LLM_MAX_TOKENS` | int | `32768` | 单次 LLM 调用的最大输出 tokens（范围 1–131072） |
| `TIMEOUT` | `LLM_TIMEOUT` | int | `300` | LLM 请求超时秒数（范围 5–600） |
| `RETRY_COUNT` | `LLM_RETRY_COUNT` | int | `3` | LLM 请求重试次数（范围 0–10） |
| `MAX_ITERATIONS` | `MAX_ITERATIONS` | int | `50` | 工具调用最大循环次数（范围 1–500） |
| `MAX_CONTEXT_TOKENS` | `MAX_CONTEXT_TOKENS` | int | `8000` | 上下文裁剪阈值，超过时移除最早非 system 消息（范围 256–131072） |
| `MEMORY_MAX_HISTORY` | `MEMORY_MAX_HISTORY` | int | `100` | 长期记忆最大保留条数（范围 0–10000） |

> `TEMPERATURE`、`MAX_TOKENS`、`TIMEOUT`、`RETRY_COUNT` 的 Python 变量名带有 `LLM_` 前缀，但配置键名和环境变量名**不含** `LLM_` 前缀。

### Python 代码中的访问方式

```python
from fp_core import config

max_iterations = config.MAX_ITERATIONS
max_context = config.MAX_CONTEXT_TOKENS
max_history = config.MEMORY_MAX_HISTORY
```

## 路径配置（XDG 标准）

路径遵循 [XDG Base Directory 规范](https://specifications.freedesktop.org/basedir-spec/latest/)，由 `XDG_CONFIG_HOME` 和 `XDG_DATA_HOME` 环境变量控制。

```python
_XDG_CONFIG_HOME = os.environ.get("XDG_CONFIG_HOME",
                     os.path.join(os.path.expanduser("~"), ".config"))
USER_CONFIG_PATH = os.path.join(_XDG_CONFIG_HOME, "fp", "config.json")

_XDG_DATA_HOME = os.environ.get("XDG_DATA_HOME",
                   os.path.join(os.path.expanduser("~"), ".local", "share"))
_XDG_DATA_FP = os.path.join(_XDG_DATA_HOME, "fp")

SESSIONS_DIR = os.path.join(_XDG_DATA_FP, "sessions")
MEMORY_DIR   = os.path.join(_XDG_DATA_FP, "memory")
TASKS_FILE   = os.path.join(_XDG_DATA_FP, "tasks.json")
PROMPTS_DIR  = os.path.join(os.path.dirname(__file__), "prompts")
```

| 路径常量 | 默认值 | 用途 |
|---------|--------|------|
| `USER_CONFIG_PATH` | `~/.config/fp/config.json` | 用户 JSON 配置文件 |
| `SESSIONS_DIR` | `~/.local/share/fp/sessions/` | 会话文件存储目录 |
| `MEMORY_DIR` | `~/.local/share/fp/memory/` | 长期记忆存储目录 |
| `TASKS_FILE` | `~/.local/share/fp/tasks.json` | 任务追踪数据文件 |
| `PROMPTS_DIR` | 源码所在目录的 `prompts/` 子目录 | 系统提示词模板目录 |

> `SESSIONS_DIR`、`MEMORY_DIR`、`TASKS_FILE` 位于 `XDG_DATA_HOME` 下（可变数据），`USER_CONFIG_PATH` 位于 `XDG_CONFIG_HOME` 下（用户配置）。

## 显示样式配置

通过 `~/.config/fp/config.json` 的 `display_styles` 和 `display_truncation` 字段配置终端输出。

### display_styles

为每种输出类型定义颜色和字体效果：

```json
{
    "display_styles": {
        "info":         {"color": "green"},
        "error":        {"color": "bright_red", "bold": true},
        "warning":      {"color": "yellow"},
        "yellow_bold":  {"color": "yellow", "bold": true},
        "llm_thought":  {"color": "magenta", "dim": true},
        "llm_tool":     {"color": "yellow"}
    }
}
```

支持的样式属性：`color`（颜色名）、`bold`（加粗）、`dim`（暗淡）、`italic`（斜体）。

| 样式名 | 默认效果 | 用途 |
|--------|---------|------|
| `info` | green | 操作反馈/成功消息 |
| `item` | 无色 | 列表子条目 |
| `hint` | 无色 | 引导/用法提示 |
| `error` | bright_red + bold | 错误消息 |
| `warning` | yellow | 警告消息 |
| `yellow_bold` | yellow + bold | 强调信息 |
| `llm_thought` | magenta + dim | LLM 思考过程 |
| `llm_tool` | yellow | 工具调用信息 |

### 核心函数

```python
def get_display_style(name: str) -> dict:
    """从 JSON 获取样式，回退到无色"""
def apply_style(text: str, name: str) -> str:
    """应用 ANSI 转义序列包裹文本，颜色不支持时返回原文本"""
def truncate(text: str, name: str) -> str:
    """按名称对应的截断长度截断；-1 不截断，截断时追加 … <+N chars>"""
def get_display_truncation(name: str) -> int:
    """获取指定名称的截断长度"""
def color_supported() -> bool:
    """FORCE_COLOR→强制; NO_COLOR→禁用; 否则检测 isatty()"""
```

### display_truncation

```json
{
    "display_truncation": {
        "tool_result": 500,
        "code_block": 200
    }
}
```

截断长度设为 `-1` 表示不截断。截断时在末尾追加 `… <+N chars>` 标记。

### 支持的颜色名称

`black`, `red`, `green`, `yellow`, `blue`, `magenta`, `cyan`, `white`, `bright_red`, `bright_green`, `bright_yellow`, `bright_cyan`, `bright_white`, `default`（无颜色）。

## 配置检查

```python
def check_llm_config() -> bool:
    """检查 LLM 配置是否完整。返回 True 表示可用，False 表示存在致命问题。"""
```

检查项：
1. `LLM_API_KEY` 是否为空
2. `LLM_API_BASE_URL` 是否为空
3. `LLM_MODEL` 是否为空
4. 汇总 Schema 验证发现的 `_validation_issues` 数量

任意一项不通过即返回 `False`，并打印带解决建议的警告信息。

## init_config 初始化

```python
def init_config(path: str | None = None):
    """初始化用户配置文件，如果文件已存在则不覆盖。"""
```

```python
def get_default_config() -> dict:
    """返回包含默认值的 dict，可用于生成 config.json。"""
```

执行流程：
1. 默认写入 `~/.config/fp/config.json`（可指定 `path` 覆盖）
2. 文件已存在则跳过，不覆盖
3. 不存在则调用 `get_default_config()` 生成带 `display_styles` 的完整配置，以 `json.dump(indent=2)` 写入

可通过命令行触发：`python -m fp_core.config --init`
