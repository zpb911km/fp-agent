# 记忆系统

## 什么是记忆

记忆系统提供**跨会话持久化**存储能力。普通的会话上下文在每次交互结束后就会丢失，而记忆可以将关键信息保存下来，在未来的会话中重新读取。

**存储位置**

```
data/memory/{name}.md
```

每个记忆对应一个 Markdown 文件，文件名即记忆名称。

---

## 记忆格式

每个记忆文件使用 **YAML frontmatter + Markdown 正文** 的格式。

**YAML frontmatter 字段**

| 字段 | 说明 |
|------|------|
| `name` | 记忆唯一标识名（英文短词） |
| `description` | 一句话描述，用于检索 |
| `type` | 记忆类型（user/project/feedback/reference） |
| `created` | 创建时间戳 |

**完整示例**

```markdown
---
name: user_role
description: 用户的角色和偏好信息
type: user
created: 2026-06-10T15:30:00
---

用户是一名全栈开发者，主要使用 Python 和 JavaScript。
偏好简洁的代码风格，注重文档质量。
不喜欢过度工程化的解决方案。
```

## 管理记忆

### 保存记忆

通过 `memory_save()` 工具保存记忆。

```python
memory_save(
    name="project_goal",
    type="project",
    description="当前项目的核心目标",
    content="开发一个 AI Agent 框架，支持技能插件和会话管理"
)
```

### 读取记忆

通过 `memory_read()` 工具读取记忆。

```python
# 列出所有记忆（不传参数）
memory_read()

# 按关键词搜索
memory_read(query="project")
memory_read(query="user preference")
```

- 空查询（不传参数）：列出全部记忆
- 传关键词：按关键词搜索，多个关键词以空格分隔，使用 AND 逻辑匹配

### 命令查看

在交互界面中使用 `/memory` 命令查看记忆列表。

```
> /memory
📋 记忆列表（3 条）

  user_role          user     用户的角色和偏好信息
  project_goal       project  当前项目的核心目标
  feedback_pref      feedback 用户对回复风格的偏好
```

## 记忆的类型

| 类型 | 说明 | 典型用途 |
|------|------|----------|
| `user` | 用户信息 | 角色、偏好、常用工作流 |
| `project` | 项目状态 | 项目目标、架构决策、进度 |
| `feedback` | 反馈偏好 | 用户对回复风格、详细程度的偏好 |
| `reference` | 参考资料 | 常用命令、代码片段、配置模板 |

选择正确的类型有助于在搜索时快速过滤，也便于后续分析和管理。
