# 技能系统 —— 导入、加载与覆写机制

技能（Skill）是注入到 LLM 系统提示词中的可插拔能力模块。每个技能是一个 Markdown 文件，通过 YAML frontmatter 定义元数据，正文为技能描述文本。

系统通过"描述索引 + 按需加载"的架构，在系统提示词中只注入技能名称和简短描述（≤80 字符），大幅节省 token 开销。

---

## 数据模型

技能数据使用 `dict[str, Skill]` 存储，`Skill` 为 `@dataclass`：

```python
@dataclass
class Skill:
    name: str                     # 唯一标识名
    title: str                    # 展示标题
    description: str              # 简要描述
    content: str = ""             # 技能正文（Markdown）
    category: str = "general"     # 分类
    version: str = "1.0"          # 版本号
    priority: int = 5             # 优先级（值越大越靠前）
    metadata: dict = field(default_factory=dict)  # 扩展元数据
```

`priority` 决定技能在提示词中的排列顺序，值越大越靠前。

## SkillLoader 双层扫描加载

`SkillLoader` 是技能的加载器，位于 `fp_core/skills/loader.py`。

### 扫描目录

| 目录 | 路径 | 说明 |
|------|------|------|
| 内置技能 | `loader.py` 所在目录 (`fp_core/skills/`) | 随项目发布的核心技能 |
| 用户技能 | `~/.local/share/fp/skills/`（XDG 标准） | 用户自定义，可覆写内置技能 |

用户目录路径遵循 XDG 标准，可通过 `$XDG_DATA_HOME` 环境变量覆盖。

### 加载流程

```python
load_all()
  ├── self.skills.clear()
  ├── _load_from_dir(BUILTIN_SKILLS_DIR)   # ① 先加载内置技能
  └── _load_from_dir(USER_SKILLS_DIR)       # ② 后加载用户技能
```

扫描使用 `os.listdir` + `fname.endswith(".md")` 过滤，文件按名称排序后依次写入 dict：

```python
for filename in sorted(os.listdir(directory)):
    if not filename.endswith(".md"):
        continue
    skill = self._load_skill_file(...)
    if skill:
        self.skills[skill.name] = skill  # 后写入覆盖先写入
```

### 同名覆写

由于 dict 键值唯一，**后加载的天然覆盖先加载的**。扫描顺序是 **内置 → 用户**，因此同名技能的用户版本自动覆盖内置版本。这是用户自定义和定制技能的机制。

## 技能文件解析

### YAML frontmatter 模式

技能文件标准格式为 **YAML frontmatter + Markdown 正文**：

```markdown
---
name: web_search
title: 网络搜索专家
description: 通过搜索获取最新网络信息
category: tool
version: 1.2
priority: 10
---

## 网络搜索技能

当需要获取实时或最新的信息时，使用 web_search 工具搜索网络。
```

解析使用正则表达式提取 frontmatter：

```python
match = re.match(r"^---\n(.*?)\n---\n(.*)$", content, re.DOTALL)
```

#### 手写 YAML 解析器

`_parse_yaml()` 是**手写解析器**（非 `yaml.safe_load()`），避免引入外部依赖。支持：
- **键值对**：`key: value`
- **自动去除外层引号**：`"value"` 或 `'value'` → `value`
- **单行列表**：`[a, b, c]`
- **多行列表**：

```yaml
tags:
  - tool
  - network
```

#### frontmatter 字段映射

| frontmatter 字段 | Skill 字段 | 默认值 | 必填 |
|---|---|---|---|
| `name` | `.name` | — | ✅ |
| `title` | `.title` | — | ✅ |
| `description` | `.description` | — | ✅ |
| `category` | `.category` | `"general"` | ❌ |
| `version` | `.version` | `"1.0"` | ❌ |
| `priority` | `.priority` | `5` | ❌ |

### 无 frontmatter 降级

文件没有 `---...---` 时自动降级处理：

| 字段 | 推导规则 |
|------|----------|
| `name` | 文件名去掉 `.md` |
| `title` | 文件名，`_` 转空格 + `.title()` |
| `description` | 正文前 200 字符 |
| `content` | 全文 |
| `category` | `"general"` |
| `priority` | `5` |

示例：`skills/my_custom_tool.md` → `name="my_custom_tool"`, `title="My Custom Tool"`。

### 错误处理

单个文件加载失败只打印警告，不中断整体流程：

```python
except Exception as e:
    print(f"[SkillLoader] Failed to load {filepath}: {e}")
    return None
```

## 模板变量替换

技能内容加载后，先替换模板变量再解析 frontmatter：

```python
@staticmethod
def _substitute_templates(text: str) -> str:
    replacements = {
        "{{path_to_agent_file}}": AGENT_ENTRY,
    }
    for placeholder, value in replacements.items():
        text = text.replace(placeholder, value)
    return text
```

当前支持的变量：

| 模板变量 | 替换值 | 用途 |
|----------|--------|------|
| `{{path_to_agent_file}}` | `fp_core/cli.py` 绝对路径 | self_modification 技能中用于启动新实例测试 |

`AGENT_ENTRY` 值在 `loader.py` 中动态计算：

```python
AGENT_ENTRY = os.path.join(
    os.path.dirname(os.path.dirname(os.path.abspath(__file__))),
    "cli.py"
)
```

## 技能注入系统提示词流程

```
Agent.__init__()
  └── self._prompter = PromptBuilder()
        └── PromptBuilder() 默认调用 create_skill_loader()
              └── SkillLoader().load_all()

PromptBuilder.build_system_prompt()
  ├── load_agent_prompt()                 # 基础 Agent 提示词
  ├── skill_loader.get_all_prompt_text()  # 技能提示词（仅名称+描述）
  ├── 当前时间/路径/用户信息              # 运行时状态
  └── 长期记忆索引                        # 记忆索引
```

### get_all_prompt_text() 输出格式

系统提示词中**只注入技能名称 + 简短描述（≤80 字符）**，不注入技能正文：

```
## 可用技能
- **子代理派遣** (`subagent`): 将多步骤、文件密集、分析型任务委派给子代理...
- **语法检查** (`python_syntax_check`): 使用 Bash 调用 Python 的 ast 模块...

需要使用时，我会主动加载完整技能内容。
```

排序规则：按 `priority` **降序**（高优先级在前）。

这是"描述索引 + 按需加载"架构，技能正文只在 LLM 需要时由工具调用读取，大幅节省 token。

## 热重载

```python
def reload(self):
    self.load_all()  # 复用完整流程：清空→内置→用户→同名覆盖
```

通过 `PromptBuilder` 暴露：

```python
prompter.reload_skills()  # → 返回当前技能总数
```

注意：`reload()` 只更新 `SkillLoader` 内部 dict，**不自动重建系统提示词**。需手动调用 `build_system_prompt()` 使新技能生效。

## 全局单例 vs 独立实例

### 全局单例（旧架构，仍保留）

```python
# loader.py 底部
skill_loader = SkillLoader()
skill_loader.load_all()
```

模块导入时自动执行，其他模块通过以下方式访问：

```python
from fp_core.skills.loader import skill_loader
```

### 独立实例工厂（新架构，推荐）

```python
def create_skill_loader() -> SkillLoader:
    loader = SkillLoader()
    loader.load_all()
    return loader
```

`PromptBuilder` 和 `Agent` 默认使用独立实例模式：

```python
# agent.py
self._prompter = prompt_builder or PromptBuilder()

# prompt_builder.py
if skill_loader is not None:
    self._skill_loader = skill_loader
else:
    from fp_core.skills.loader import create_skill_loader
    self._skill_loader = create_skill_loader()
```

代码注释明确标注"不再使用全局单例"。独立实例适用于多 Agent 实例场景，彼此技能状态完全隔离。

## 用户技能编写指南

### 注册新技能

在用户技能目录下创建 `.md` 文件即可：

```
~/.local/share/fp/skills/your_skill.md
```

推荐使用完整的 YAML frontmatter 格式：

```markdown
---
name: your_skill
title: 你的技能名称
description: 简短描述，将在系统提示词中展示
category: tool
version: 1.0
priority: 5
---

## 技能正文

Markdown 格式的技能描述和使用说明...
```

### 覆写内置技能

如需自定义某个内置技能的行为，在用户目录下创建**同名 `name` 字段**的文件即可。用户版本自动覆盖内置版本。

### 关键设计决策

| 决策 | 理由 |
|------|------|
| 手写 YAML 解析器而非 `yaml.safe_load()` | 避免引入外部依赖，技能 frontmatter 格式简单，手写解析更轻量 |
| 按 priority 降序排列 | 高优先级技能（核心工具）排在前面，降低 LLM 遗漏的风险 |
| 只注入标题+描述，不注入正文 | 大幅节省系统提示词 token，正文按需加载 |
| 后加载覆盖先加载 | 天然的覆写机制，无需额外配置；用户技能流程与内置完全一致 |
| 独立实例而非全局单例 | 支持多 Agent 实例各自持有独立的技能状态，互不干扰 |
