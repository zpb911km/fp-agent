# 插件系统

插件系统是基于「钩子（Hook）」的生命周期扩展机制。插件可以挂载到 Agent 运行的各个阶段，实现横切关注点的注入。

---

## Plugin 基类

基类位于 `plugins/base/plugin.py`：

```python
from abc import ABC, abstractmethod
from typing import Optional

class Plugin(ABC):
    name: str                                    # 插件唯一标识名
    version: str                                 # 语义化版本号

    def __init__(self, config: dict): ...        # 接收配置
    def on_register(self, lifecycle): ...         # 注册时调用，在此注册钩子
    def on_unregister(self): ...                  # 注销时调用，清理资源
    def enable(self): ...                         # 启用
    def disable(self): ...                        # 禁用
    def is_enabled(self) -> bool: ...             # 当前启用状态
```

## PluginRegistry

全局插件注册中心，管理全部插件的生命周期：

```python
class PluginRegistry:
    def scan(self): ...          # 扫描 plugins/ 目录，发现插件文件
    def register(self, plugin): ...   # 注册单个插件实例
    def unregister(self, name): ...   # 注销指定插件
    def get(self, name) -> Plugin: ...# 按名称获取插件
    def list_plugins(self): ...       # 列出所有已注册插件
```

---

## 文件即开关规则

`PluginRegistry.scan()` 按以下规则扫描 `fp_core/plugins/` 目录：

| 文件名 | 行为 |
|--------|------|
| `name.py` | ✅ 加载 |
| `name.py.disabled` | ❌ 跳过（不加载） |
| `_name.py` | ❌ 跳过（下划线前缀视为内部模块） |
| `base.py` | ❌ 跳过（基类模块） |

可以通过 `mv plugin.py plugin.py.disabled` 临时禁用，或 `mv plugin.py.disabled plugin.py` 重新启用，**无需修改代码或配置**。

## 插件文件格式

每个插件是一个继承 `Plugin` 的 Python 类。文件放置在 `fp_core/plugins/` 目录下。

**完整示例：**

```python
"""plugins/my_plugin.py"""
from fp_core.plugins.base.plugin import Plugin

class MyPlugin(Plugin):
    name = "my_plugin"
    version = "1.0.0"

    def __init__(self, config: dict):
        super().__init__(config)
        self.counter = 0

    def on_register(self, lifecycle):
        # 注册多个钩子，可按优先级排序
        lifecycle.register("ON_MESSAGE", self.on_message, priority=10, name=self.name)
        lifecycle.register("ON_AFTER_RESPONSE", self.after_response, priority=5, name=self.name)

    def on_unregister(self):
        self.counter = 0

    async def on_message(self, ctx, **kwargs):
        self.counter += 1
        ctx.user_input = f"[插件#{self.counter}] {ctx.user_input}"
        return ctx

    async def after_response(self, ctx, **kwargs):
        ctx.response += f"\n\n--- 已处理 {self.counter} 条消息 ---"
        return ctx
```

## 生命周期钩子注册

`lifecycle.register(hook, func, priority, name)` 注册钩子处理函数。

**参数说明：**
- `hook: str` —— 钩子名称（见下方列表）
- `func` —— 处理函数，签名见下
- `priority: int` —— 优先级，越小越先执行（默认 100）
- `name: str` —— 插件标识名

**处理函数签名：**

```python
handler(ctx: HookContext, **kwargs) -> Optional[HookContext]
```

- 接收 `HookContext` 和额外参数
- 返回修改后的 `ctx`，或返回 `None`（不修改）
- `async` 或同步均可，框架自动适配

### 完整 19 个钩子列表

| # | 钩子名 | 触发时机 | 关键 kwargs |
|---|--------|----------|-------------|
| 1 | `ON_STARTUP` | Agent 启动完成 | `agent` |
| 2 | `ON_SHUTDOWN` | Agent 关闭前 | `agent` |
| 3 | `ON_MESSAGE` | 收到用户消息 | `user_input` |
| 4 | `ON_BEFORE_RESPONSE` | LLM 回复前 | `messages` |
| 5 | `ON_AFTER_RESPONSE` | LLM 回复后 | `response`, `agent` |
| 6 | `ON_TOOL_CALL` | LLM 请求调用工具 | `tool_name`, `tool_args` |
| 7 | `ON_TOOL_RESULT` | 工具执行完成 | `tool_name`, `result` |
| 8 | `ON_ERROR` | 发生未捕获异常 | `error`, `traceback` |
| 9 | `ON_SESSION_SWITCH` | 切换会话 | `old_session`, `new_session` |
| 10 | `ON_SESSION_SAVE` | 保存会话 | `session` |
| 11 | `ON_SESSION_LOAD` | 加载会话 | `session` |
| 12 | `ON_SKILL_LOAD` | 技能加载完成 | `skills` |
| 13 | `ON_SKILL_RELOAD` | 技能重载完成 | `skills` |
| 14 | `ON_COMMAND` | 执行命令前 | `command`, `arg` |
| 15 | `ON_COMMAND_RESULT` | 命令执行后 | `command`, `result` |
| 16 | `ON_MODEL_SWITCH` | 切换模型 | `old_model`, `new_model` |
| 17 | `ON_CONFIG_CHANGE` | 配置变更 | `key`, `old_value`, `new_value` |
| 18 | `ON_FORK` | 分叉会话 | `source_session`, `forked_session` |
| 19 | `ON_RESET` | 会话重置 | `session` |

## NotificationPlugin 参考实现

`plugins/notification.py` 提供了一套完整的桌面+声音通知方案。

**设计要点：**

```python
class NotificationPlugin(Plugin):
    name = "notification"
    version = "1.0.1"
```

**注册的钩子：**
- `ON_ERROR` —— 发生错误时弹出错误通知
- `ON_BEFORE_RESPONSE` —— LLM 回复前清空旧通知
- `ON_SHUTDOWN` —— 关闭时发送退出通知

**快捷创建函数：**

```python
def create_notification_plugin(
    desktop: bool = True,         # 启用桌面通知（KDE Connect）
    sound: bool = False,           # 启用声音通知
    rate_limit: float = 5.0,       # 最小通知间隔（秒）
    hooks: list[str] = None,       # 要注册的钩子列表
    max_desktop_len: int = 100,    # 桌面通知最大字符数（超出截断）
) -> NotificationPlugin:
    ...
```

用户可在 `~/.config/fp/config.json` 的 `plugins.notification` 段中传入这些参数。

```json
{
  "plugins": {
    "notification": {
      "desktop": true,
      "sound": false,
      "rate_limit": 3.0,
      "max_desktop_len": 80
    }
  }
}
```
