# 工具系统 —— 导入、注册与覆写机制（全异步重构版）

工具系统是 Agent 与外部世界交互的桥梁。LLM 通过 Function Calling 声明需要调用的工具，由 `ToolRegistry` 统一调度执行并返回结果。

---

## 架构概览

```
ToolRegistry.__init__()
  ├── _load_core()     → core.py 硬绑定（4 个核心工具）
  └── _load_plugins()  → plugins/ 自动扫描（内置 → 用户）
       ├── _load_from_dir(builtin_dir, "fp_core.tools.plugins")
       └── _load_from_dir(user_dir)  # 同名覆盖
```

**核心原则：**

- **bash, read_file, write_file, edit_file** —— 保持直接绑定（`core.py`），不可插件化
- 其他工具通过 `plugins/*_plugin.py` 插件机制动态加载

---

## 核心工具（不可插件化）

4 个核心工具在 `tools/core.py` 中定义，**不可通过用户插件卸载或覆盖**。

| 工具名 | 参数 | 说明 |
|--------|------|------|
| `bash` | `command: str` | 执行 shell 命令，超时 300s |
| `read_file` | `file_path: str, offset?: int, limit?: int` | 读取文件，支持行范围 |
| `write_file` | `file_path: str, content: str` | 创建或覆写文件 |
| `edit_file` | `file_path: str, old_string: str, new_string: str` | 精确字符串替换（首次出现） |

`tools/core.py` 中每个工具定义包含：

1. **OpenAI schema** —— 描述参数结构
2. **异步执行函数** —— `async def execute_xxx(params) -> str`
3. **统一分发入口** —— `execute_core_tool(name, params)` 负责路由

```python
# tools/core.py 结构示意
TOOL_DEFINITIONS = [
    {
        "type": "function",
        "function": {
            "name": "bash",
            "description": "执行 shell 命令",
            "parameters": {
                "type": "object",
                "properties": {
                    "command": {"type": "string", "description": "命令内容"}
                },
                "required": ["command"]
            }
        }
    },
    # ...read_file, write_file, edit_file
]

async def execute_core_tool(name: str, params: dict) -> str:
    """路由到对应实现"""
```

## 插件工具的两种模式

### 模式 A：单工具模式（11/12 个插件使用）

**约定：** `PLUGIN_DEFINITION` + `execute(params) -> str`

```python
PLUGIN_DEFINITION = {
    "type": "function",
    "function": {
        "name": "web_search",
        "description": "快速搜索...",
        "parameters": { ... }
    }
}

async def execute(params: dict) -> str:
    query = params["query"]
    ...
    return str(results)
```

内部存储 key = `plugin_name`（文件名去掉 `_plugin.py`）

### 模式 B：多工具模式（目前只有 kdeconnect_plugin 使用）

**约定：** `PLUGIN_DEFINITIONS`（列表）+ `TOOL_MAP`（`{tool_name: executor}`）

```python
PLUGIN_DEFINITIONS = [
    {"type": "function", "function": {"name": "kdeconnect_notify", ...}},
    {"type": "function", "function": {"name": "kdeconnect_share_text", ...}},
    {"type": "function", "function": {"name": "kdeconnect_send_screenshot", ...}},
]

TOOL_MAP = {
    "kdeconnect_notify": _exec_notify,
    "kdeconnect_share_text": _exec_share_text,
    "kdeconnect_send_screenshot": _exec_screenshot,
}
```

内部存储 key = `f"{plugin_name}/{tool_name}"`（如 `"kdeconnect/kdeconnect_notify"`）

**两种模式的判断顺序：** 先检查多工具模式（`PLUGIN_DEFINITIONS` + `TOOL_MAP`），再检查单工具模式（`PLUGIN_DEFINITION` + `execute`）。

## ToolRegistry 类

```python
class ToolRegistry:
    def get_all_definitions(self) -> list[dict]:
        """合并核心 + 插件工具定义（核心在前，插件在后）"""

    async def execute(self, tool_name: str, params: dict) -> Any:
        """路由到核心或插件执行器"""
```

### 执行路由逻辑

```python
async def execute(self, tool_name, params):
    # 1. 核心工具直接路由
    if tool_name in {"bash", "read_file", "write_file", "edit_file"}:
        return await self._core_executor(tool_name, params)
    # 2. 遍历 _plugins values() 匹配 definition["function"]["name"]
    for plugin_data in self._plugins.values():
        if def_name == tool_name:
            return await plugin_data["executor"](params)
    raise ValueError(f"未知工具: {tool_name}")
```

## 同名覆写机制

### 加载顺序驱动的覆盖

```python
def _load_plugins(self):
    self._load_from_dir(builtin_dir, ...)  # 先内置
    self._load_from_dir(user_dir)           # 后用户 → 同名覆盖
```

内置插件先写入 `_plugins` dict，用户插件后写入 —— **同名 key 直接覆盖**。

### 覆写能力边界

| 类别 | 可覆写？ | 原因 |
|------|----------|------|
| 核心工具（bash/read_file/write_file/edit_file） | ❌ 不可 | 通过独立 `_core_executor` 分发 |
| 内置插件 | ✅ 可 | 用户目录同名文件覆盖 |
| 用户自定义新工具 | ✅ 可 | 无白名单限制 |

### 实际案例：web_search 覆写

用户 `~/.local/share/fp/tools/plugins/web_search_plugin.py` 与内置同名，其 `PLUGIN_DEFINITION` 中的 description 增加了"如果需要综合分析多篇文章的内容，请用 smart_web_search"的提示，这是降级到 smart_web_search 的引导。

## 用户目录加载

```python
# 内置目录（包路径导入）
module = importlib.import_module(f".plugins.{plugin_name}", package="fp_core.tools")

# 用户目录（文件路径导入）
spec = importlib.util.spec_from_file_location(plugin_name, filepath)
module = importlib.util.module_from_spec(spec)
spec.loader.exec_module(module)
```

- **用户目录：** `~/.local/share/fp/tools/plugins/`（XDG 标准）
- **扫描条件：** 文件名以 `_plugin.py` 结尾，按字母序排序

## 全局单例 vs 独立实例

### 全局单例

```python
registry = ToolRegistry()  # 模块加载时自动初始化
```

`dispatch()`、`execute_tool()` 以及快捷函数共享此实例。

### 独立实例工厂（新架构，推荐）

```python
def create_registry() -> ToolRegistry:
    return ToolRegistry()  # 全新实例，重新扫描
```

`ToolExecutor` 默认使用 `create_registry()`，支持多 Agent 隔离。

---

## ToolExecutor 封装

```python
class ToolExecutor:
    def __init__(self, registry=None):
        self._registry = registry or create_registry()  # 默认独立实例

    def get_definitions(self) -> list[dict]:
        return self._registry.get_all_definitions()

    async def execute(self, tool_call: dict) -> str:
        name = tool_call["function"]["name"]
        args = json.loads(tool_call["function"]["arguments"])
        result = await self._registry.execute(name, args)
        return str(result)
```

**三层封装：**

1. **输入适配** —— 从 LLM tool_call 中提取 name + args
2. **输出适配** —— 任意类型转为 str
3. **错误隔离** —— 异常由框架统一捕获处理

## 插件开发规范

### 单工具模式示例

```python
"""tools/plugins/my_tool_plugin.py"""

PLUGIN_DEFINITION = {
    "type": "function",
    "function": {
        "name": "my_tool",
        "description": "示例工具——将输入翻转",
        "parameters": {
            "type": "object",
            "properties": {
                "input": {
                    "type": "string",
                    "description": "要翻转的文本"
                }
            },
            "required": ["input"]
        }
    }
}

async def execute(params: dict) -> str:
    """推荐异步实现；错误时 raise Exception。"""
    text = params["input"]
    return text[::-1]
```

### 多工具模式示例

```python
"""tools/plugins/my_multi_plugin.py"""

PLUGIN_DEFINITIONS = [
    {
        "type": "function",
        "function": {
            "name": "tool_a",
            "description": "...",
            "parameters": { ... }
        }
    },
    {
        "type": "function",
        "function": {
            "name": "tool_b",
            "description": "...",
            "parameters": { ... }
        }
    },
]

TOOL_MAP = {
    "tool_a": _exec_tool_a,
    "tool_b": _exec_tool_b,
}

async def _exec_tool_a(params: dict) -> str:
    ...

async def _exec_tool_b(params: dict) -> str:
    ...
```

### 规则

- `execute` / 各 executor 必须返回 `str`（LLM 接收的是字符串）
- 推荐异步（`async def`），同步会自动包装
- 发生错误时 `raise Exception`，由框架统一捕获并返回错误消息
