# 会话管理

## 自动保存

每次用户交互完成后，系统自动将会话保存到 `~/.local/share/fp/sessions/` 目录。

**文件命名格式**

```
s_YYMMDD_HHMMSSffffff.jsonl
```

例如：`s_260610_153022123456.jsonl`

**文件结构**

- 第 1 行是 meta 行（JSON 格式），包含字段：
  - `id`：会话唯一标识
  - `created`：创建时间戳
  - `updated`：最后更新时间戳
  - `summary`：会话摘要（自动生成）
  - `message_count`：消息总条数
- 第 2 行起，每行一条消息记录，按正序存储，格式为 JSON

示例：

```jsonl
{"id": "s_260610_153022123456", "created": "2026-06-10T15:30:22", "updated": "2026-06-10T16:45:10", "summary": "讨论了配置文件修改和技能热重载", "message_count": 12}
{"role": "user", "content": "你好，请帮我修改配置", "timestamp": "2026-06-10T15:30:22"}
{"role": "assistant", "content": "好的，请告诉我需要修改什么配置", "timestamp": "2026-06-10T15:30:25"}
```

## 会话列表

使用 `/resume` 命令（不带参数）可交互式列出所有历史会话。

**交互示例**

```
> /resume
📋 历史会话列表：

  #0  s_260610_153022123456  2026-06-10 15:30  讨论了配置文件修改和技能热重载
  #1  s_260609_090011654321  2026-06-09 09:00  编写单元测试用例
  #2  s_260608_142500987654  2026-06-08 14:25  分析系统日志异常

输入序号 (#0, #1...) 或会话 ID 恢复，输入 q 取消
>
```

## 恢复会话

支持三种恢复方式：

### 命令行启动时恢复

```bash
# 恢复最近的会话
python3 cli.py -r

# 恢复指定会话
python3 cli.py -r s_260610_153022123456
```

### 运行中恢复

在交互界面中使用 `/resume` 命令：

```
# 恢复最近的会话
> /resume latest

# 恢复指定会话
> /resume s_260610_153022123456
```

恢复后会将会话历史载入上下文，可继续之前的对话。

## 分支 Fork

`/fork` 命令基于当前上下文创建一个新的会话分支。

**行为说明**

- 新会话继承当前上下文中的所有消息
- 原会话保持不变，系统自动为其生成摘要标签
- 新会话获得新的会话 ID，从 fork 点继续对话

**示例**

```
> /fork
🔄 会话已分叉！
   原会话: s_260610_153022123456（已保存归档）
   新会话: s_260610_170000789012（继续对话）
```

## 会话文件管理

### 文件位置

所有会话文件存储在 `~/.local/share/fp/sessions/` 目录下。

### 备份

直接复制整个目录即可备份：

```bash
cp -r data/sessions/ backup/sessions_20260610/
```

### 查看

```bash
# 列出所有会话文件
ls -lh data/sessions/

# 查看会话元信息
head -1 data/sessions/s_260610_153022123456.jsonl | python3 -m json.tool

# 查看会话摘要
head -1 data/sessions/s_260610_153022123456.jsonl | python3 -c "import sys,json; print(json.load(sys.stdin)['summary'])"
```

### 清理空会话

系统内置 `delete_empty_sessions` 技能，用于扫描并清理记录为空（仅有 meta 行、无实际消息）的会话文件。

```bash
# 预览模式（仅列出，不删除）
# 调用 delete_empty_sessions(dry_run=True)

# 执行清理
# 调用 delete_empty_sessions(dry_run=False)
```

清理依据逐行计数而非依赖 meta 字段，防止因 meta 数据错误导致误判。
