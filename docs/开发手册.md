## 1. 项目概述

### 1.1 定位

**Five Pebbles** 是一个 CLI 交互式 AI Agent，灵感来源于游戏 *Rain World* 中的迭代器角色。它是一个自持、可扩展的半自主 AI，具备：

- 通过 LLM 进行自然语言对话
- 工具调用（执行 Shell 命令、读写文件、网络搜索、子 Agent 等）
- 技能系统（可热重载的能力描述）
- 插件系统（生命周期钩子驱动的功能扩展）
- 会话管理（多会话、断点续传、上下文中止/压缩/回退）
- 自我修改能力（修改自身源码并测试验证）

### 1.2 技术栈

| 组成部分 | 技术选型 | 版本要求 |
|----------|----------|----------|
| 语言 | Python | ≥ 3.10 |
| 异步框架 | asyncio | 标准库 |
| HTTP 客户端 | [httpx](https://www.python-httpx.org/) | ≥ 0.24 |
| 交互终端 | [prompt_toolkit](https://python-prompt-toolkit.readthedocs.io/) | ≥ 3.0 |
| Markdown 渲染 | [rich](https://rich.readthedocs.io/)（可选） | ≥ 13.0 |
| 全宽字符检测 | [wcwidth](https://pypi.org/project/wcwidth/)（可选） | — |
| 搜索引擎 | httpx + HTML 解析 (Bing/DuckDuckGo) | 内置，用于 `web_search` |

### 1.3 快速启动

```bash
# 安装依赖
pip install httpx prompt_toolkit rich wcwidth

# 配置 API Key（3 种方式选一）
# 方式 1：编辑 config.json（推荐）
# 方式 2：export LLM_API_KEY="sk-xxx"
# 方式 3：使用 --init 生成默认配置
python cli.py --init

# 启动交互式会话
python cli.py

# 单次消息模式
python cli.py -m "帮我查看 current 目录下的 Python 文件"

# 续上一个会话
python cli.py -r

# 续指定会话
python cli.py -r s_260606_160000000000
```

### 3.2 数据流

```
用户输入
    │
    ▼
┌─────────────────────────────────────────────────┐
│ Agent.process(user_input)                        │
│                                                  │
│  1. 检查是否为命令 (/xxx) → 交给 commands        │
│                                                  │
│  2. 添加 user msg 到上下文                       │
│     │  lifecycle.emit(ON_MESSAGE_RECEIVED)       │
│     ▼                                            │
│  3. LLM 调用（while 循环）                        │
│     │  lifecycle.emit(ON_BEFORE_LLM_CALL)        │
│     ▼                                            │
│  ┌───────────────────────────────────┐            │
│  │ LLM Client (httpx 异步)           │            │
│  │  POST /chat/completions           │            │
│  │  ← assistant msg (content/tool)   │            │
│  └───────────────┬───────────────────┘            │
│                  ▼                                │
│     lifecycle.emit(ON_AFTER_LLM_CALL)             │
│                                                  │
│  4. 检查 tool_calls                              │
│     ┌─── 有工具调用?                             │
│     │   ├── yes → 逐个执行工具                    │
│     │   │    lifecycle.emit(ON_TOOL_CALL)        │
│     │   │    → tools.dispatch(name, **args)      │
│     │   │    lifecycle.emit(ON_TOOL_RESULT)      │
│     │   │    → 添加 result 到上下文 → 回步骤 3    │
│     │   │    (循环检测，最多 MAX_ITERATIONS 次)   │
│     │   │                                        │
│     │   └── no → 跳出循环，生成最终回复           │
│     ▼                                            │
│  5. 裁剪上下文 (trim_context)                    │
│     lifecycle.emit(ON_CONTEXT_UPDATE)            │
│                                                  │
│  6. 保存到会话文件 (session.save_context)         │
│     lifecycle.emit(ON_BEFORE_RESPONSE)           │
│     ▼                                            │
│  返回 Response(content)                           │
└─────────────────────────────────────────────────┘
                    │
                    ▼
              用户看到回复
```

### 3.3 生命周期系统

LifecycleManager 提供了 19 个生命周期钩子（`LifecycleHook` 枚举），覆盖 Agent 的完整执行路径。

#### 钩子列表

| 阶段 | 钩子名 | 触发时机 | 传递数据 |
|------|--------|----------|----------|
| **初始化** | `ON_INIT` | Agent 首次初始化 | — |
| | `ON_CONFIG_LOADED` | 配置加载完成 | — |
| **消息** | `ON_MESSAGE_RECEIVED` | 收到用户消息 | `content` |
| | `ON_MESSAGE_PARSE` | 消息解析 | — |
| | `ON_MESSAGE_FILTER` | 消息过滤 | — |
| **执行** | `ON_BEFORE_THINK` | 思考前 | — |
| | `ON_THINK` | 思考中 | — |
| | `ON_AFTER_THINK` | 思考后 | — |
| **LLM** | `ON_BEFORE_LLM_CALL` | LLM 调用前 | — |
| | `ON_LLM_CALL` | LLM 调用中 | — |
| | `ON_AFTER_LLM_CALL` | LLM 调用后 | `has_tool_calls`, `tool_names`, `content` |
| **响应** | `ON_BEFORE_RESPONSE` | 生成回复前 | `content` |
| | `ON_RESPONSE` | 生成回复中 | — |
| | `ON_AFTER_RESPONSE` | 生成回复后 | — |
| **工具** | `ON_TOOL_SELECT` | 工具选择 | `tools` |
| | `ON_TOOL_CALL` | 工具调用前 | `tool_name`, `tool_args` |
| | `ON_TOOL_RESULT` | 工具调用后 | `tool_name`, `result` |
| | `ON_TOOL_ERROR` | 工具错误 | `error` |
| **上下文** | `ON_CONTEXT_UPDATE` | 上下文更新 | `msg_count` |
| | `ON_CONTEXT_READ` | 上下文读取 | — |
| **错误** | `ON_ERROR` | 发生错误 | `error` |
| | `ON_RETRY` | 重试 | — |
| **资源** | `ON_SHUTDOWN` | 关闭 | — |
| | `ON_CLEANUP` | 清理资源 | — |

#### HookContext 传递

```python
@dataclass
class HookContext:
    hook: LifecycleHook          # 当前钩子
    data: Dict[str, Any]         # 数据字典（由 emit 的 kwargs 填充）
    metadata: Dict[str, Any]     # 元数据
    stop_propagation: bool       # 设为 True 可阻止后续钩子执行
    error: Optional[Exception]   # 捕获钩子执行中的异常
```

关键规则：
- 钩子按 `priority` 升序执行（0 最先，999 最后）
- 同步/异步自动适配（检查 `iscoroutinefunction`）
- `stop_propagation=True` 中断后续钩子链
- 钩子可以返回修改过的 `HookContext` 对象传递给下一个钩子

#### emit / emit_sync

```python
# 异步触发（Agent 主流程使用）
await lifecycle.emit(LifecycleHook.ON_TOOL_CALL, tool_name="bash", tool_args="ls -la")

# 同步触发（非异步环境）
lifecycle.emit_sync(LifecycleHook.ON_ERROR, error="Connection refused")
```

## 4. 核心模块 API

### 4.1 Agent 类 (`core/agent.py`)

```python
class Agent:
    def __init__(self, enable_log: bool = False, resume: Optional[str] = None)
    #     enable_log: 开启内部日志
    #     resume: None=新会话, "auto"=续最近, "s_xxx"=续指定

    # ── 主入口 ──────────────────────────────────
    async def process(self, user_input: str) -> Response
    #     处理用户输入（全异步）
    #     自动处理：命令判断 → LLM 调用 → 工具循环 → 上下文管理 → 持久化

    async def shutdown(self)
    #     关闭 Agent（保存上下文 + LLM 生成总结 + 统计面板 + 清理）

    # ── 技能 ─────────────────────────────────────
    def reload_skills(self) -> bool
    #     热重载所有技能文件，返回是否成功

    def list_skills(self) -> str
    #     返回技能列表文本

    # ── 会话 ─────────────────────────────────────
    def list_sessions(self) -> str
    def switch_session(self, sid: str) -> bool
    def clear_session(self)

    # ── 中断 ─────────────────────────────────────
    def cancel(self)
    #     信号处理器回调，设置中断标志

    # ── 命令 ─────────────────────────────────────
    @property
    def COMMANDS(self) -> Dict[str, str]
    #     返回 {"/cmd_name": "描述"} 字典

    async def handle_command(self, cmd_line: str) -> bool

    # ── 内置属性 ─────────────────────────────────
    # self.client      — LLM 客户端
    # self.lifecycle   — LifecycleManager
    # self.plugins     — PluginRegistry
    # self.session     — SessionManager
    # self.model       — 当前模型名
    # self._context    — 对话上下文列表
```

### 4.3 LifecycleManager (`core/lifecycle.py`)

```python
class LifecycleManager:
    def __init__(self, enable_log: bool = False)

    def register(self, hook: LifecycleHook, func: Callable,
                 priority: int = 100, name: Optional[str] = None)
    #     注册钩子处理器，按 priority 升序执行

    def unregister(self, hook: LifecycleHook, name: str) -> bool

    async def emit(self, hook: LifecycleHook,
                   context: Optional[HookContext] = None, **kwargs) -> HookContext

    def get_hooks(self, hook: LifecycleHook) -> List[str]
    def clear(self, hook: Optional[LifecycleHook] = None)
    def get_stats(self) -> Dict[str, int]
```

### 4.4 LLM Client (`core/llm_client.py`)

自实现的 OpenAI HTTP 客户端，使用 `httpx.AsyncClient`，无需安装 `openai` SDK。

```python
class Client:
    def __init__(self, api_key: str,
                 base_url: str = "https://api.deepseek.com/v1",
                 timeout: int = 300)

    async def close(self)

    # 通过 chat.completions.create 使用：
    client.chat.completions.create(...)
```

**`create()` 参数：**

```python
async def create(
    model: str,                       # 模型名
    messages: list[dict],             # 消息列表
    stream: bool = False,             # 是否流式
    temperature: Optional[float],     # 温度
    max_tokens: Optional[int],        # 最大 token
    tools: Optional[list[dict]],      # OpenAI function calling schema
    extra_body: Optional[dict],       # 额外请求体参数
) -> Union[CompletionResponse, SSEIterator]
```

**返回类型：**

- `stream=False`：`CompletionResponse` — 非流式完整响应
  - `response.choices[0].message` — `Message` 对象（含 `content`, `tool_calls`, `reasoning_content`）
  - `response.choices[0].message.tool_calls` — `List[ToolCall]`
- `stream=True`：`SSEIterator` — 可 `async for` 迭代 `Chunk` 对象
  - 自动处理 `<think>...</think>` 标签提取（将思考内容放入 `reasoning_content`）

**think 标签提取：**

```python
# 自动处理两种思考内容返回格式：
# 1. 原生 reasoning_content 字段（首选）
# 2. <think>...</think> 标签嵌入在 content 中（fallback）
# SSEIterator 和 Message 构造函数都自动处理了这种 fallback
```

### 4.5 Plugin / PluginRegistry (`plugins/base/plugin.py`)

```python
class Plugin(ABC):
    name: str = "base_plugin"
    version: str = "1.0.0"

    def __init__(self, config: Optional[PluginConfig] = None)
    def on_register(self, lifecycle: LifecycleManager)    # 子类必须实现
    def on_unregister(self)
    def enable(self) / def disable(self)
    @property
    def is_enabled(self) -> bool

class PluginRegistry:
    def __init__(self, lifecycle: LifecycleManager, plugin_dir: str)

    def scan(self, plugin_dir: str) -> List[str]     # 自动扫描目录
    def register(self, plugin: Plugin) -> Plugin     # 手动注册
    def unregister(self, name: str) -> Optional[Plugin]
    def get(self, name: str) -> Optional[Plugin]
    def list_plugins(self) -> List[str]
```

**文件即开关规则：**

## 5. 工具开发指南

工具分为两类：**核心工具**（直接绑定在 `tools/core.py`）和 **插件工具**（自动扫描 `tools/plugins/` 目录）。

### 5.1 核心工具（不可插件化）

4 个核心工具必须保持直接绑定：

| 工具名 | 说明 | 参数 |
|--------|------|------|
| `bash` | 执行 Shell 命令 | `command` |
| `read_file` | 读文件 | `file_path`, `offset?`, `limit?` |
| `write_file` | 写文件 | `file_path`, `content` |
| `edit_file` | 精确替换 | `file_path`, `old_string`, `new_string` |

这些工具在 `tools/core.py` 中定义，包含：
1. OpenAI schema 定义（`CORE_TOOL_DEFINITIONS`）
2. 异步执行函数（`_execute_bash`/`_execute_read_file`/...）
3. 统一分发入口（`execute_core_tool`）

### 5.2 插件工具开发

要添加新工具，只需在 `tools/plugins/` 目录下创建 `*_plugin.py` 文件。

#### 文件格式要求

```python
# tools/plugins/my_new_tool_plugin.py
"""
My New Tool 插件描述（同步/异步均可）
"""

from typing import Any, Dict

# ── 必须导出：OpenAI function calling schema ──
PLUGIN_DEFINITION = {
    "type": "function",
    "function": {
        "name": "my_new_tool",                              # 工具名（小写+下划线）
        "description": "工具的详细描述，LLM 据此决定何时调用",  # 描述
        "parameters": {
            "type": "object",
            "properties": {
                "param1": {
                    "type": "string",                        # 参数类型
                    "description": "参数描述",
                },
                "param2": {
                    "type": "integer",
                    "description": "参数描述",
                },
            },
            "required": ["param1"],                          # 必填参数
        },
    },
}


# ── 必须导出：执行函数（异步推荐，同步也可）──
async def execute(params: Dict[str, Any]) -> str:
    """
    执行工具逻辑

    Args:
        params: 参数字典，键名对应 PLUGIN_DEFINITION 中定义的参数

    Returns:
        字符串形式的执行结果
    """
    param1 = params.get("param1", "")
    param2 = params.get("param2", 0)

    # ... 工具逻辑 ...

    return f"执行结果: {result}"
```

#### 关键规则

1. **必须导出** `PLUGIN_DEFINITION` 和 `execute`（名必须完全一致）
2. **文件名必须**以 `_plugin.py` 结尾（如 `my_new_tool_plugin.py`）
3. **异步推荐**：`async def execute(...)`，同步也可用
4. **返回字符串**：所有结果强制 `str(result)` 返回
5. **错误处理**：`raise Exception` 会被上层捕获为 `"❌ 工具执行失败: ..."`

#### 注册流程（自动）

```python
# tools/__init__.py 中的 ToolRegistry._load_plugins()
def _load_plugins(self):
    plugins_dir = "tools/plugins/"
    for fname in os.listdir(plugins_dir):
        if not fname.endswith("_plugin.py"):
            continue
        module = importlib.import_module(f".plugins.{plugin_name}", package="tools")
        if hasattr(module, 'PLUGIN_DEFINITION') and hasattr(module, 'execute'):
            self._plugins[plugin_name] = {
                'definition': module.PLUGIN_DEFINITION,
                'executor': module.execute,
            }
```

```python
# skills/loader.py 中的 SkillLoader
class SkillLoader:
    def load_all(self):
        for filename in os.listdir("skills/"):
            if not filename.endswith('.md'):
                continue
            skill = self._load_skill_file(os.path.join("skills/", filename))
            if skill:
                self.skills[skill.name] = skill

    def get_all_prompt_text(self) -> str:
        # 按 priority 降序排序，拼接为提示词
        parts = ["\n\n## 可用技能\n"]
        for skill in sorted(self.skills.values(), key=lambda s: -s.priority):
            parts.append(f"### {skill.title} ({skill.name})\n")
            parts.append(f"{skill.content}\n\n")
        return '\n'.join(parts)
```

## 8. 插件开发指南

插件是 Python 模块，继承 `Plugin` 基类，放置在 `plugins/` 目录。

### 8.1 插件文件格式

```python
# plugins/my_plugin.py
"""
我的自定义插件
"""

from typing import Optional

from plugins.base.plugin import Plugin, PluginConfig
from core.lifecycle import LifecycleHook, HookContext, LifecycleManager


class MyPlugin(Plugin):
    """我的自定义插件"""

    name = "my_plugin"          # 必填：插件标识名
    version = "1.0.0"          # 推荐：版本号

    def __init__(self, config: Optional[PluginConfig] = None):
        super().__init__(config)
        # 自定义初始化逻辑

    def on_register(self, lifecycle: LifecycleManager):
        """注册生命周期钩子（必实现）"""
        # 注册一个钩子
        lifecycle.register(
            LifecycleHook.ON_MESSAGE_RECEIVED,  # 钩子类型
            self._on_message,                    # 处理函数
            priority=100,                        # 优先级（可选）
            name=f"{self.name}_on_message",      # 名称（可选）
        )

        # 也可以注册多个钩子
        lifecycle.register(
            LifecycleHook.ON_BEFORE_RESPONSE,
            self._on_before_response,
            priority=50,
        )

    def on_unregister(self):
        """插件卸载时清理（可选）"""
        pass

    async def _on_message(self, ctx: HookContext, **kwargs):
        """钩子处理函数"""
        content = kwargs.get("content", "")
        print(f"[MyPlugin] 收到消息: {content[:50]}...")

        # 可以修改上下文
        ctx.data["filtered"] = True
        return ctx

    async def _on_before_response(self, ctx: HookContext, **kwargs):
        """响应前处理"""
        content = kwargs.get("content", "")
        # 修改内容（可选）
        # ctx.data["content"] = "修改后的内容"
        return ctx
```

### 8.4 NotificationPlugin 示例

```python
# plugins/notification.py — 完整参考实现

from plugins.base.plugin import Plugin, PluginConfig
from core.lifecycle import LifecycleHook, HookContext, LifecycleManager

class NotificationPlugin(Plugin):
    name = "notification"
    version = "1.0.1"

    # 标志位
    desktop: bool = True
    sound: bool = True
    rate_limit: float = 2.0

    def on_register(self, lifecycle: LifecycleManager):
        for hook in [LifecycleHook.ON_ERROR,
                     LifecycleHook.ON_BEFORE_RESPONSE,
                     LifecycleHook.ON_SHUTDOWN]:
            lifecycle.register(
                hook,
                self._make_handler(hook),
                priority=50,
                name=f"notify_{hook.name}"
            )

    def _make_handler(self, hook):
        async def handler(ctx, **kwargs):
            if not self._enabled:
                return
            self._notify(hook, **kwargs)
        return handler
```

## 11. 中断处理机制

### 11.1 设计目标

跨平台（Unix/Windows）可靠的中断处理，用户按 Ctrl+C 时优雅响应。

### 11.2 架构

```
Ctrl+C (SIGINT)
    │
    ▼
signal.signal(SIGINT, _raw_sigint_handler)  ← CLI 级处理器
    │
    ├──→ asyncio.all_tasks().cancel()       ← Unix 主线程
    │       │                                   注入 CancelledError
    │       ▼
    │   各个 asyncio 任务捕获 CancelledError
    │
    └──→ _interrupted_flag = True           ← 跨平台回退（Windows）
            │
            ▼
         Agent._check_interrupted()         ← 检查点
             │
             ▼
         raise asyncio.CancelledError
```

### 11.3 双重检查机制

```python
# cli.py — 安装信号处理器
def _raw_sigint_handler(signum, frame):
    # 方式 1（Unix 主线程）：直接取消所有 asyncio 任务
    try:
        for task in asyncio.all_tasks():
            task.cancel()
    except (RuntimeError, ValueError):
        pass

    # 方式 2（跨平台回退）：设置全局标志
    try:
        import core.agent as _agent_mod
        _agent_mod._interrupted_flag = True
    except Exception:
        pass

# core/agent.py — 检查点
class Agent:
    def _check_interrupted(self):
        global _interrupted_flag
        if _interrupted_flag:
            _interrupted_flag = False
            self._interrupted = False
            self._processing = False
            raise asyncio.CancelledError("用户中断")
        if self._interrupted:
            self._interrupted = False
            self._processing = False
            raise asyncio.CancelledError("用户中断")
```

### 11.4 检查点位置

`_check_interrupted()` 在以下关键位置被调用（通过 `CancelledError` 被 `try/except` 捕获）：

1. **LLM 调用中**：`_stream_chat()` 的 httpx 请求可被取消
2. **工具执行中**：`_execute_tool()` 中的 bash 子进程被 `proc.kill()`
3. **主循环中**：`process()` 的 `while True` 循环检查点

### 11.5 中断恢复策略

```python
# 当 LLM 调用中途被中断时：
# 1. 保留已生成的内容（partial content）
# 2. 如果有 tool_calls 但未执行，标记为"用户中断"
assistant_msg.pop("_interrupted", False)
if interrupted and "tool_calls" in assistant_msg:
    tc_names = [tc["function"]["name"] for tc in assistant_msg["tool_calls"]]
    note = f"\n\n[用户中断 — 计划调用的工具: {', '.join(tc_names)}，请求已被用户打断]"
    assistant_msg["content"] = (content + note) if content else note.strip()
    del assistant_msg["tool_calls"]

# 当工具执行中被中断时：
# 1. 当前工具标记为"工具调用失败：用户中断"
# 2. 剩余工具标记为"未执行"
# 3. 退出工具循环，保留已有上下文
```

### 11.6 `_processing` 标志

```python
# Agent 在 LLM 调用前设置 self._processing = True
# 在 LLM 调用后恢复 self._processing = False
# 信号处理器可通过检查此标志判断当前是否处于可中断状态
```

## 12. 配置系统

### 12.1 三级优先级

```
config.json  (最高) →  环境变量  →  代码默认值  (最低)
```

```python
def _value(key: str, default: Any = None) -> Any:
    # 1. JSON 中显式指定且不为 null
    if key in _json_cfg and _json_cfg[key] is not None:
        return _json_cfg[key]
    # 2. 环境变量（自动类型转换）
    env_val = os.getenv(key)
    if env_val is not None:
        if key in ("TEMPERATURE",):
            return float(env_val)
        if key in ("MAX_TOKENS", "MAX_ITERATIONS", "TIMEOUT", ...):
            return int(env_val)
        return env_val
    # 3. 默认值
    return default
```
