# 插件系统

## 架构

插件系统采用**生命周期钩子驱动**的架构。系统在运行的关键节点触发钩子，已注册的插件按优先级依次执行对应的钩子函数。

**工作流程**

```
[系统事件] ──触发──→ [钩子调度器] ──依次调用──→ [已注册的插件钩子函数]
```

---

## 生命周期钩子

系统共定义了 19 个生命周期钩子，按阶段分组。

| 阶段 | 钩子名 | 触发时机 |
|------|--------|----------|
| **初始化** | `on_start` | 系统启动时 |
| | `on_ready` | 系统初始化完成时 |
| | `on_config_loaded` | 配置文件加载完成时 |
| **消息处理** | `on_user_message` | 收到用户消息时 |
| | `on_before_process` | 开始处理用户消息前 |
| | `on_after_process` | 完成处理后 |
| **思考** | `on_before_think` | LLM 开始思考前 |
| | `on_after_think` | LLM 思考完成后 |
| **LLM 调用** | `on_before_llm_call` | 调用 LLM API 前 |
| | `on_after_llm_call` | LLM API 返回结果后 |
| | `on_llm_error` | LLM API 调用出错时 |
| **响应** | `on_before_response` | 生成最终回复前 |
| | `on_after_response` | 最终回复发送后 |
| **工具** | `on_before_tool_call` | 调用工具前 |
| | `on_after_tool_call` | 工具执行完毕后 |
| **上下文** | `on_context_loaded` | 上下文加载完成时 |
| | `on_context_compacted` | 上下文压缩完成后 |
| **错误** | `on_error` | 发生未捕获异常时 |
| **资源** | `on_shutdown` | 系统关闭时 |

## 内置插件

### 通知插件

在交互式模式下，通知插件在以下时机触发桌面通知和提示音：

- `on_after_response`：Agent 回复完成后，发出通知提醒用户查看
- `on_error`：发生错误时，发出告警通知
- `on_llm_error`：API 调用失败时，发出失败通知

通知通过 `kdeconnect_notify` 技能可同步推送到手机。

## 插件开发概览

### 开发步骤

1. 继承 `Plugin` 基类
2. 实现 `on_register` 方法
3. 在 `on_register` 中注册需要的生命周期钩子
4. 将插件文件放入 `plugins/` 目录

### 最小示例

```python
from .base import Plugin

class HelloWorldPlugin(Plugin):
    """一个简单的示例插件"""

    def on_register(self):
        """注册钩子"""
        self.register_hook("on_start", self.on_start)
        self.register_hook("on_user_message", self.on_user_message)

    async def on_start(self):
        """系统启动时执行"""
        print("👋 HelloWorld 插件已加载")

    async def on_user_message(self, message: str):
        """收到用户消息时执行"""
        if "hello" in message.lower():
            print("🌍 用户说了 hello！")
```

### 钩子函数签名

所有钩子函数均为异步函数，接收与触发时机相关的上下文参数。具体参数请参考基类中的类型注解。
