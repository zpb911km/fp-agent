## 分层架构

Five Pebbles 采用四层架构设计，从入口到数据逐层收敛，职责清晰。

```
┌─────────────────────────────────────────────────────────┐
```

**各层职责**：

| 层 | 持有者 | 职责 |
|----|--------|------|
| 入口层 | `agent.py` + `cli.py` | 参数解析、交互循环、命令分发 |
| 核心层 | `core/agent.py` 中的 `Agent` 类 | 编排主循环，持有所有服务实例 |
| 插件/工具层 | `tools/` + `plugins/` + `skills/` | 工具注册与执行、生命周期钩子、技能热加载 |
| 数据层 | `data/` 目录 | 会话持久化、长期记忆、任务追踪 |

## 数据流

一次用户输入的完整处理链路：

```
用户输入 "查询今天的天气"
       │
       ▼
┌───────────────────────────────────┐
│ 1. 命令判断 (cli.py / Agent)      │
│    ┌─────────────────────────┐    │
│    │ 以 / 开头 → handle_command│   │
│    │ 否则 → process()        │    │
│    └─────────────────────────┘    │
└──────────┬────────────────────────┘
           │ (非命令)
           ▼
┌───────────────────────────────────┐
│ 2. 生命周期: ON_MESSAGE_FILTER    │
│    (transform: 可修改/阻止消息)    │
└──────────┬────────────────────────┘
           ▼
┌───────────────────────────────────┐
│ 3. LLM 调用循环 (while)           │
│    ┌─────────────────────────┐    │
│    │ 构建上下文 (system+历史) │    │
│    │         │                │    │
│    │ 生命周期: ON_BEFORE_LLM  │    │
│    │  (可修改 messages/tools)  │    │
│    │         │                │    │
│    │ LLMClient.chat.complet-  │    │
│    │ ions.create() → Response │    │
│    │         │                │    │
│    │ 生命周期: ON_AFTER_LLM   │    │
│    │  (可修改 response)       │    │
│    │         │                │    │
│    │ ┌─── 有 tool_calls? ──┐ │    │
│    │ │ 是          │ 否     │ │    │
│    │ ▼             ▼       │ │    │
│    │ 工具执行    返回内容   │ │    │
│    │ (ToolExec)  退出循环 ─┘ │    │
│    │    │                    │    │
│    │ 生命周期: ON_TOOL_*    │    │
│    │    │                    │    │
│    │ 结果追加到上下文,继续   │    │
│    └─────────────────────────┘    │
└──────────┬────────────────────────┘
           ▼
┌───────────────────────────────────┐
│ 4. 上下文裁剪 (context pruning)   │
│    超过 MAX_CONTEXT_TOKENS 时     │
│    移除最早的非 system 消息       │
└──────────┬────────────────────────┘
           ▼
┌───────────────────────────────────┐
│ 5. 生命周期: ON_BEFORE_RESPONSE   │
│    (transform: 可修改最终回复)     │
└──────────┬────────────────────────┘
           ▼
┌───────────────────────────────────┐
│ 6. 持久化                          │
│    session.save_context()         │
│    session.save_message()         │
│    更新文件内嵌 meta (message_cnt)│
└──────────┬────────────────────────┘
           ▼
┌───────────────────────────────────┐
│ 7. 输出响应到终端                  │
│    display.*()                    │
└───────────────────────────────────┘
```

**关键设计点**：

- LLM 调用是一个 **while 循环**，每次迭代检查返回的 `tool_calls`。有工具调用则执行工具、结果追加到上下文、继续调用 LLM；无工具调用则退出循环。
- 生命周期钩子在每个关键节点被触发，插件可以观察或转换数据流。
- 上下文裁剪是避免 token 爆炸的最后防线。

## 生命周期系统

### LifecycleManager

`core/lifecycle.py` 中的 `LifecycleManager` 实现了事件驱动的钩子系统，是整个插件体系的基石。

### 19 个钩子 (按阶段分组)

#### 初始化阶段

| 钩子 | 类型 | 触发时机 | 事件上下文 |
|------|------|----------|-----------|
| `ON_INIT` | observe | Agent 初始化完成 | — |
| `ON_CONFIG_LOADED` | observe | 配置加载完成 | — |

#### 消息处理阶段

| 钩子 | 类型 | 触发时机 | 事件上下文 |
|------|------|----------|-----------|
| `ON_MESSAGE_FILTER` | **transform** | 用户消息过滤/修改 | `MessageFilterEvent` |
| `ON_MESSAGE_RECEIVED` | observe | 消息已接收（仅通知） | — |

#### 执行阶段

| 钩子 | 类型 | 触发时机 | 事件上下文 |
|------|------|----------|-----------|
| `ON_BEFORE_LLM_CALL` | **transform** | LLM 调用前 — 可修改 messages/tools，或取消调用 | `BeforeLLMCallEvent` |
| `ON_AFTER_LLM_CALL` | **transform** | LLM 返回后 — 可修改 response，或阻止工具执行 | `AfterLLMCallEvent` |

#### 响应阶段

| 钩子 | 类型 | 触发时机 | 事件上下文 |
|------|------|----------|-----------|
| `ON_BEFORE_RESPONSE` | **transform** | 返回响应前 — 可修改 content | `BeforeResponseEvent` |

#### 工具阶段

| 钩子 | 类型 | 触发时机 | 事件上下文 |
|------|------|----------|-----------|
| `ON_TOOL_SELECT` | observe | 工具已选择 | — |
| `ON_TOOL_CALL` | observe | 工具即将调用 | `ToolCallEvent` |
| `ON_TOOL_RESULT` | observe | 工具调用完成 | `ToolResultEvent` |
| `ON_TOOL_ERROR` | observe | 工具错误 | — |

#### 上下文阶段

| 钩子 | 类型 | 触发时机 | 事件上下文 |
|------|------|----------|-----------|
| `ON_CONTEXT_UPDATE` | observe | 上下文已更新 | `ContextUpdateEvent` |

#### 错误阶段

| 钩子 | 类型 | 触发时机 | 事件上下文 |
|------|------|----------|-----------|
| `ON_ERROR` | observe | 发生错误 | `ErrorEvent` |

#### 资源阶段

| 钩子 | 类型 | 触发时机 | 事件上下文 |
|------|------|----------|-----------|
| `ON_SHUTDOWN` | observe | 关闭中 | — |
| `ON_CLEANUP` | observe | 清理资源 | — |

### HookContext 传递机制

```python
@dataclass
class HookContext:
    hook: LifecycleHook          # 当前钩子枚举
    data: dict                   # 事件负载数据
    metadata: dict               # 元数据
    stop_propagation: bool       # 停止后续钩子传播
    error: Exception | None      # 执行错误
    hook_type: str               # "observe" | "transform"
```

**传递规则**：

1. **priority 升序执行** — 优先级数值越小越先执行（priority=0 最优先）
2. **stop_propagation** — 设为 `True` 后停止后续钩子
3. **transform 型钩子** — 异常会向上传播（抛出），导致流程中断
4. **observe 型钩子** — 异常被隔离捕获，不中断流程

### 同步/异步自动适配

```python
# 钩子函数可以是同步或异步，emit() 自动检测
if asyncio.iscoroutinefunction(func):
    result = await func(context, **context.data)
else:
    result = func(context, **context.data)
```

### API

```python
# 注册钩子
lifecycle.register(
    hook=LifecycleHook.ON_BEFORE_LLM_CALL,
    func=my_hook_handler,
    priority=100,         # 越小越优先
    name="my_handler",    # 可选，用于 unregister
)

# 注销钩子
lifecycle.unregister(LifecycleHook.ON_BEFORE_LLM_CALL, "my_handler")

# 异步触发
context = await lifecycle.emit(LifecycleHook.ON_BEFORE_LLM_CALL, context)

# 同步触发（非异步环境）
context = lifecycle.emit_sync(LifecycleHook.ON_SHUTDOWN, context)
```

**typde 事件上下文约定**：transform 型钩子使用 `@dataclass` 定义输入/输出字段，插件通过修改字段值来影响流程。
